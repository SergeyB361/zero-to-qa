# Неделя 6, День 1 — HTTP и REST

## Что такое HTTP
HTTP — это протокол, по которому клиент и сервер обмениваются запросами и ответами.

## Что обычно важно в запросе
- метод: `GET`, `POST`, `PUT`, `PATCH`, `DELETE`
- URL
- headers
- body

## Что обычно важно в ответе
- статус-код
- headers
- тело ответа

## Частые методы
- `GET` — получить данные
- `POST` — создать данные
- `PUT` — полностью обновить
- `PATCH` — частично обновить
- `DELETE` — удалить

## REST в простом виде
REST — это стиль построения API, где ресурсы обычно представлены через понятные URL и стандартные HTTP-методы.

Пример:
- `GET /users` — получить список пользователей
- `GET /users/1` — получить одного пользователя
- `POST /users` — создать пользователя

## Статус-коды
- `200 OK` — запрос успешен
- `201 Created` — ресурс создан
- `204 No Content` — успешный ответ без тела
- `400 Bad Request` — ошибка в запросе
- `401 Unauthorized` — нет авторизации
- `403 Forbidden` — доступ запрещён
- `404 Not Found` — ресурс не найден
- `500 Internal Server Error` — ошибка сервера

## Практические правила
- всегда смотри не только на тело ответа, но и на статус-код
- одинаковый URL может вести себя по-разному в зависимости от метода
- для QA важно понимать, что именно должен делать каждый endpoint
