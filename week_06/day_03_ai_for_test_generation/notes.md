# Неделя 6, День 3 — AI для генерации тестов

## Что AI умеет генерировать
- unit-тесты
- API-тесты
- UI-тесты
- фикстуры
- тестовые данные

## Где это полезно
- когда нужен быстрый старт
- когда есть понятная функция или контракт API
- когда нужен черновик структуры теста

## Главный риск
AI пишет тесты правдоподобно, но нередко:
- выдумывает поля ответа
- делает неверные assertions
- не понимает реальный бизнес-смысл

## Как использовать правильно
Проси:
- сначала кратко объяснить логику теста
- затем сгенерировать код
- потом отдельно перечислить предположения и риски

## Самая полезная схема работы
1. выбрать реальную функцию из репозитория
2. попросить AI сгенерировать тест
3. проверить assumptions
4. исправить ошибки вручную
5. запустить тест

Без шага 4 и 5 AI-генерация тестов почти не имеет ценности.

## Хорошие кандидаты в этом курсе
- функции из калькулятора
- функции из JSON parser мини-проекта
- простые методы `User` и `Admin`
