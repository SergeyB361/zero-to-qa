# Неделя 6, День 3 — Первые API-тесты

## Идея API-теста
API-тест отправляет запрос и проверяет, что ответ соответствует ожиданиям.

## Что обычно проверяют
- статус-код
- структуру JSON
- конкретные поля
- бизнес-смысл ответа

## Пример
```python
def test_get_user():
    response = requests.get("https://reqres.in/api/users/2")
    assert response.status_code == 200
    data = response.json()
    assert data["data"]["id"] == 2
```

## Хороший порядок проверки
1. проверить статус-код
2. распарсить JSON
3. проверить ключевые поля
4. при необходимости проверить дополнительные детали

## Практические правила
- не делай слишком много разных проверок в одном тесте без причины
- имя теста должно объяснять, какой endpoint и какой сценарий проверяется
- сначала нужны понятные и надёжные проверки, а не сложность ради сложности
