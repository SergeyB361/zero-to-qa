# Неделя 5, День 2 — Первые тесты PyTest

## Что такое PyTest
`pytest` — популярный фреймворк для запуска автотестов на Python.

## Почему он удобен
- простая запись тестов
- понятные `assert`
- много готовых возможностей: фикстуры, параметризация, плагины

## Базовые правила
- тестовые файлы обычно называются `test_*.py`
- тестовые функции обычно называются `test_*`
- запуск: `pytest`

## Структура простого теста
```python
def test_sum():
    assert 2 + 2 == 4
```

## Как выглядит минимальный сценарий
1. есть код, который нужно проверить
2. есть тестовый файл
3. в тестовом файле лежит функция `test_*`
4. запускаешь `pytest`

## Что нужно для старта
- установить `pytest`
- создать файл с кодом
- создать файл тестов
- запустить `pytest`

## Практические правила
- тест должен проверять одну понятную вещь
- имена тестов должны объяснять сценарий
- тесты должны запускаться предсказуемо и без ручных действий
