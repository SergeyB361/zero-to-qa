# Неделя 5, День 3 — Assertions

## Что такое assertion
`assert` проверяет, что условие истинно. Если нет, тест падает.

## Что обычно проверяют
- равенство значений
- вхождение элемента
- тип результата
- текст ошибки
- булевый результат

## Примеры
```python
def test_sum():
    assert 2 + 2 == 4

def test_name():
    assert "qa" in "qa automation"
```

## Проверка исключений
Для ожидаемых ошибок часто используют `pytest.raises`.

```python
import pytest

def test_divide_by_zero():
    with pytest.raises(ZeroDivisionError):
        10 / 0
```

## Почему это важно
Хорошая проверка должна быть:
- понятной
- точной
- связанной с целью теста

Слабая проверка — это когда тест запускается, но почти ничего не доказывает.

## Практические правила
- проверяй не всё подряд, а то, что действительно важно для сценария
- если ожидается ошибка, проверяй именно ошибку, а не случайный побочный эффект
- один тест лучше с одной сильной идеей, чем с множеством случайных `assert`
