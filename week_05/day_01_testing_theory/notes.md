# Неделя 5, День 1 — Теория тестирования

## Что такое тестирование
Тестирование помогает проверить, что программа работает так, как ожидается, и не ломается после изменений.

## Зачем это нужно QA Automation
Автотесты нужны не ради самих тестов. Их цель:
- быстро находить регрессии
- проверять критичные сценарии стабильно
- давать команде уверенность перед релизом

## Базовые уровни тестов
- `unit` — проверяем маленький участок кода: функцию, метод, класс
- `integration` — проверяем, как несколько частей работают вместе
- `e2e` — проверяем пользовательский сценарий целиком

## Чем они отличаются
- `unit` быстрые и изолированные
- `integration` проверяют связи между частями системы
- `e2e` ближе всего к реальному поведению пользователя, но они медленнее и сложнее

## Пирамида тестов
Идея пирамиды:
- больше unit-тестов
- меньше integration-тестов
- ещё меньше e2e-тестов

Почему так:
- unit-тесты быстрее
- их проще поддерживать
- e2e-тесты дороже и медленнее

## Что обычно проверяет QA Automation
- бизнес-логику
- интеграции API
- UI-сценарии
- регресс важных флоу

## Баг-репорт
Хороший баг-репорт обычно содержит:
- заголовок
- шаги воспроизведения
- ожидаемый результат
- фактический результат
- окружение
- вложения при необходимости

## Практические правила
- Не всё нужно проверять e2e-тестом.
- Старайся выбирать самый дешёвый уровень теста, который всё ещё ловит нужную проблему.
- Один тест должен иметь понятную цель.
