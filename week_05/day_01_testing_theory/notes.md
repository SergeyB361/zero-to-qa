# Неделя 5, День 1 — Теория тестирования

## Что такое тестирование
Тестирование помогает проверить, что программа работает так, как ожидается, и не ломается после изменений.

## Базовые уровни тестов
- `unit` — проверяем маленький участок кода: функцию, метод, класс
- `integration` — проверяем, как несколько частей работают вместе
- `e2e` — проверяем пользовательский сценарий целиком

## Пирамида тестов
Идея пирамиды:
- больше unit-тестов
- меньше integration-тестов
- ещё меньше e2e-тестов

Почему так:
- unit-тесты быстрее
- их проще поддерживать
- e2e-тесты дороже и медленнее

## Что обычно проверяет QA Automation
- бизнес-логику
- интеграции API
- UI-сценарии
- регресс важных флоу

## Баг-репорт
Хороший баг-репорт обычно содержит:
- заголовок
- шаги воспроизведения
- ожидаемый результат
- фактический результат
- окружение
- вложения при необходимости

Открой `practice.py` и классифицируй тестовые сценарии по уровням.
