# Неделя 5, День 6 — Запуск и отчёты

## Полезные команды PyTest
- `pytest` — запуск всех тестов
- `pytest -v` — подробный вывод
- `pytest -k name` — запуск тестов по части имени
- `pytest --tb=short` — короткий traceback

## Зачем это нужно
- быстрее запускать нужные тесты
- проще читать результат падения
- удобнее локализовать проблему

## Как читать результат запуска
Обычно тебе важно понять:
- сколько тестов прошло
- какой тест упал
- на каком `assert` произошло падение
- какой traceback показывает причину

## Allure
Allure используют для более наглядных отчётов по тестам.
На этом этапе важно понимать:
- это отдельный инструмент отчётности
- он делает тестовые результаты удобнее для чтения
- позже ты будешь использовать его глубже

## Практические правила
- не запускай всегда весь набор тестов, если проверяешь один маленький сценарий
- умей быстро сузить запуск через имя теста или файла
- сначала научись понимать обычный вывод `pytest`, потом уже красивые отчёты
