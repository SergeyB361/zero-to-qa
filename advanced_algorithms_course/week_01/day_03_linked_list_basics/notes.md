# Неделя 1, День 3 - Linked List: база

## Что такое linked list
Связный список - это структура, где каждый узел хранит:
- значение;
- ссылку на следующий узел.

В отличие от массива, элементы не обязаны лежать подряд по индексам.

## Минимальная модель узла
Обычно узел выглядит так:
- `value`
- `next`

## Как мыслить связным списком
В массиве мы обычно думаем индексами.
В linked list нужно думать переходами по ссылкам:
- текущий узел;
- следующий узел;
- конец списка = `None`.

## Где это полезно
- базовое понимание linked structures;
- задачи на reversal;
- slow/fast pointers;
- понимание, как работают некоторые структуры под капотом.

## Что неудобно в linked list
- нет быстрого доступа по индексу;
- чтобы дойти до элемента, нужно пройти по ссылкам;
- реализация обычно сложнее массива.

## Типичные ошибки
1. Потерять ссылку на следующий узел.
2. Переписать `next` раньше времени.
3. Не обрабатывать пустой список.
4. Путать голову списка и текущий указатель.

## Практическое правило
При работе со связным списком почти всегда полезно держать в голове:
- `current`
- `next_node`
- `head`
