# Неделя 2, День 1 - Heap / Priority Queue

## Что такое heap
Heap - это структура данных, которая позволяет быстро получать наименьший или наибольший элемент.

В Python стандартный `heapq` реализует **min-heap**:
- самый маленький элемент всегда наверху.

## Что такое priority queue
Priority queue - это очередь, где элементы извлекаются не просто по порядку поступления, а по приоритету.

Часто её удобно реализовать через heap.

## Базовые операции в Python
Через `heapq`:
- `heappush(heap, x)`
- `heappop(heap)`

Обе операции работают эффективно.

## Когда heap полезен
- получить минимум/максимум среди динамически меняющихся данных;
- задачи на `top-k`;
- планировщики задач;
- алгоритмы на графах;
- priority-based processing.

## Важная мысль
Heap **не делает весь массив отсортированным**.
Он гарантирует только локальное свойство:
- корень содержит минимальный элемент.

## Где это реально используют
- очередь задач с приоритетами;
- поиск нескольких лучших элементов;
- merge нескольких потоков данных;
- shortest path алгоритмы.

## Типичные ошибки
1. Думать, что heap - это просто отсортированный список.
2. Пытаться использовать heap как обычный массив с полной сортировкой.
3. Путать min-heap и max-heap.
4. Забывать, что `heapq` работает на списке in-place.

## Практическое правило
Если в задаче нужно много раз брать минимум или поддерживать множество кандидатов, подумай о heap.
