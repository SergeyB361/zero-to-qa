# Неделя 2, День 4 - BFS

## Что такое BFS
BFS = breadth-first search.

Это обход графа по слоям:
- сначала ближайшие вершины;
- потом вершины на следующем расстоянии;
- потом ещё дальше.

## Какая структура нужна
Для BFS нужна очередь.

Это естественно:
- вершины попадают в очередь,
- обрабатываются в порядке поступления.

## Где BFS особенно полезен
- кратчайший путь в невзвешенном графе;
- расстояния по уровням;
- поиск ближайшей вершины с условием;
- обход по слоям.

## Базовая идея
1. Положить стартовую вершину в очередь.
2. Отметить её как посещённую.
3. Пока очередь не пуста:
   - взять вершину из начала;
   - пройти по соседям;
   - непосещённых добавить в очередь.

## Почему нужен visited
Иначе в графе с циклами можно ходить бесконечно.

## Типичные ошибки
1. Не использовать visited.
2. Использовать стек вместо очереди.
3. Отмечать вершину посещённой слишком поздно.
4. Путать BFS и DFS.

## Практическое правило
Если задача звучит как:
- ближайший;
- минимальное число шагов;
- обход по уровням,

то сначала подумай о BFS.
