# Неделя 5, День 1 - Стек

## Что такое стек
Стек - это структура данных по принципу **LIFO**:
- last in
- first out

Последний добавленный элемент извлекается первым.

## Базовые операции
- `push` - положить элемент в стек
- `pop` - снять элемент со стека
- `peek/top` - посмотреть верхний элемент без удаления

## Как стек обычно реализуют в Python
Для учебных задач чаще всего достаточно обычного списка:

```python
stack = []
stack.append(10)  # push
value = stack.pop()  # pop
```

Это простой и нормальный вариант.

## Где стек полезен
- разворот последовательности;
- проверка скобок;
- возврат к предыдущему состоянию;
- обходы и откат шагов;
- задачи на парсинг.

## Как мыслить стеком
Если задача звучит как:
- “нужно запомнить последние открытые элементы”;
- “нужно откатываться в обратном порядке”;
- “последний пришёл - первый ушёл”,

то почти наверняка нужен стек.

## Типичные ошибки
1. Делать `pop()` из пустого стека.
2. Путать стек и очередь.
3. Искать стек там, где нужен просто список.
4. Не проверять, пуст стек или нет.

## Практическое правило
Стек хорошо подходит для задач, где важен обратный порядок обработки.
