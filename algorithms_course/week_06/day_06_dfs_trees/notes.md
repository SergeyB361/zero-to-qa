# Неделя 6, День 6 - DFS по дереву

## Что такое DFS
DFS = depth-first search.

Идея:
- идём вглубь структуры;
- потом возвращаемся назад.

Для дерева это очень естественный способ обхода.

## Варианты обхода
### Preorder
- текущий узел;
- левое поддерево;
- правое поддерево.

### Inorder
- левое поддерево;
- текущий узел;
- правое поддерево.

### Postorder
- левое поддерево;
- правое поддерево;
- текущий узел.

## Почему это важно
Разный порядок обхода даёт разный смысл результата.

Например:
- inorder часто полезен для бинарных деревьев поиска;
- preorder полезен для сериализации структуры;
- postorder полезен, когда нужно сначала обработать потомков.

## Типичные ошибки
1. Путать порядок вызовов.
2. Не обрабатывать `None`.
3. Склеивать результат в неправильном порядке.
4. Не понимать, зачем нужны разные варианты DFS.
