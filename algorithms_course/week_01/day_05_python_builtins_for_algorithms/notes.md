# Неделя 1, День 5 - Встроенные структуры Python для алгоритмов

## Зачем это нужно
Иногда задача кажется алгоритмической, но в Python уже есть встроенный инструмент, который решает её проще и чище.

Важно не только уметь писать алгоритм руками, но и уметь выбирать правильную встроенную структуру.

## Что особенно важно
### `list`
Подходит для:
- хранения последовательности;
- прохода по элементам;
- индексного доступа.

### `dict`
Подходит для:
- быстрых lookup по ключу;
- подсчёта частот;
- кэширования результатов.

### `set`
Подходит для:
- проверки уникальности;
- быстрого membership check;
- удаления дубликатов.

### `enumerate()`
Нужно, когда в цикле важны и индекс, и значение.

### `zip()`
Нужно, когда надо пройти по двум последовательностям параллельно.

### `sorted()`
Используется, когда важна сортировка, но писать её руками не нужно.

## Практическая идея
Часто правильный вопрос не “какой алгоритм написать”, а “какую структуру данных выбрать”.

## Типичные примеры
- быстро проверить, был ли элемент раньше -> `set`
- посчитать количество встреч -> `dict`
- пройти по массиву с индексами -> `enumerate`
- объединить два списка в пары -> `zip`

## Типичные ошибки
1. Использовать список там, где нужен `set`.
2. Писать ручной счётчик частот без `dict`.
3. Игнорировать `enumerate()` и хранить индекс вручную.
4. Писать лишний код там, где хватает built-in решения.
