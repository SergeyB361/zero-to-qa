# Неделя 4, День 1 - dict и set как алгоритмический инструмент

## Главная мысль
Очень многие алгоритмические задачи резко упрощаются, если выбрать правильную структуру данных.

В Python чаще всего это:
- `dict`
- `set`

Они позволяют делать быстрые проверки и накопление данных без лишнего перебора.

## Когда нужен dict
`dict` нужен, когда есть логика вида:
- ключ -> значение
- элемент -> количество
- идентификатор -> объект
- слово -> позиция

Типовые примеры:
- частоты элементов;
- быстрый доступ по id;
- кэширование результатов;
- last seen index.

## Когда нужен set
`set` нужен, когда важен сам факт присутствия элемента:
- уже видели элемент или нет;
- есть ли дубликат;
- принадлежит ли значение множеству;
- сколько уникальных элементов.

## Почему это алгоритмически важно
Если использовать список, проверка вида:

```python
if x in items:
```

может стоить `O(n)`.

Если использовать `set`, аналогичная проверка обычно близка к `O(1)`.

Это огромная разница на больших входах.

## Примеры задач, где помогает set
- найти дубликат;
- проверить уникальность;
- убрать повторяющиеся элементы;
- проверить, был ли элемент раньше.

## Примеры задач, где помогает dict
- посчитать частоты;
- собрать индекс по ключу;
- хранить пары `число -> индекс`;
- запоминать промежуточные результаты.

## Что важно понимать
`dict` и `set` хороши только для hashable-значений:
- числа;
- строки;
- кортежи из hashable-значений.

Например, список нельзя положить в `set` или использовать как ключ `dict`.

## Типичные ошибки
1. Использовать список там, где нужен быстрый lookup.
2. Забывать, что `set` не хранит порядок как основную гарантию задачи.
3. Выбирать `dict`, хотя нужен просто факт присутствия элемента, и тогда лучше `set`.
4. Пытаться использовать mutable-объекты как ключи.

## Практическое правило
Если задача сводится к вопросам:
- “видели ли мы это раньше?”
- “сколько раз встречалось значение?”
- “как быстро получить что-то по ключу?”

то почти всегда стоит подумать о `dict` или `set`.
