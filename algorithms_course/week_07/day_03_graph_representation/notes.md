# Неделя 7, День 3 - Представление графа

## Что такое граф
Граф - это набор:
- вершин;
- рёбер между вершинами.

Графы нужны, когда данные представляют связи.

## Примеры графов
- города и дороги;
- пользователи и подписки;
- страницы и ссылки;
- сервисы и зависимости.

## Как обычно представляют граф в задачах
Самый практичный способ - **adjacency list**.

Пример:

```python
graph = {
    "A": ["B", "C"],
    "B": ["D"],
    "C": [],
    "D": []
}
```

Это значит:
- из `A` можно попасть в `B` и `C`
- из `B` в `D`

## Почему adjacency list удобен
- прост в Python;
- удобно обходить соседей;
- хорошо подходит для BFS и DFS.

## Направленный и ненаправленный граф
### Направленный
Если есть ребро `A -> B`, это не значит, что есть `B -> A`.

### Ненаправленный
Связь двусторонняя.
Тогда ребро обычно добавляют в обе стороны.

## Что важно для кода
Нужно заранее понимать:
- граф направленный или нет;
- есть ли изолированные вершины;
- могут ли быть циклы;
- хранятся ли веса.

## Типичные ошибки
1. Добавить ребро только в одну сторону для ненаправленного графа.
2. Не создать вершину без соседей.
3. Путать вершины и рёбра.
4. Плохо понимать, что именно хранится в adjacency list.
