# Неделя 7, День 2 - Top-K паттерны

## Что такое top-k задача
Это задачи вида:
- найти `k` наибольших элементов;
- найти `k` наименьших;
- найти `k` самых частых;
- поддерживать текущие лучшие кандидаты.

## Почему heap здесь полезен
Если нужен не полный порядок, а только `k` лучших элементов, heap позволяет не сортировать всё заново.

## Базовая идея
### Для k наименьших
Можно:
- либо использовать `heapq.nsmallest`;
- либо поддерживать структуру кандидатов.

### Для k наибольших
Можно:
- использовать `heapq.nlargest`;
- или вручную поддерживать heap размера `k`.

## Где это полезно
- топ-частые слова;
- топ-приоритетные задачи;
- несколько лучших оценок;
- top-k search results.

## Практический вопрос
Если нужен полный порядок, можно сортировать.
Если нужен только top-k, heap часто лучше и концептуально чище.

## Типичные ошибки
1. Полностью сортировать массив там, где нужен только top-k.
2. Путать “наибольшие значения” и “самые частые”.
3. Неправильно выбирать структуру для промежуточных кандидатов.
