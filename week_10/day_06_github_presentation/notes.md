# Неделя 10, День 6 — Оформление GitHub

## Зачем это важно
GitHub-репозиторий — это часть профессиональной презентации. По нему видно не только код, но и качество инженерного подхода.

## Что обычно важно
- понятный README
- чистая структура проекта
- осмысленные коммиты
- инструкции по запуску
- понятные названия файлов и папок

## Практические правила
- хороший репозиторий должен быть понятен без устных объяснений
- README должен отвечать на вопрос: что это, как запустить, что проверяется
- оформление не должно быть декоративным, оно должно помогать

Открой `practice.py` и продумай, как сделать репозиторий понятным для ревьюера или работодателя.
