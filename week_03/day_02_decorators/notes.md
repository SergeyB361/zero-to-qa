# Неделя 3, День 2 — Декораторы: база

## Что такое декоратор
Декоратор — это функция, которая принимает другую функцию, добавляет ей поведение и возвращает новую функцию.

Простыми словами:
- была обычная функция,
- мы "обернули" её другой функцией,
- после этого при вызове выполняется дополнительный код.

## Зачем это нужно
Декораторы помогают не дублировать одинаковую логику.
Например:
- логирование вызова функции,
- измерение времени,
- проверка доступа,
- проверка аргументов.

## Базовая схема
```python
def decorator(func):
    def wrapper():
        print("before")
        func()
        print("after")
    return wrapper
```

## Как это работает
- `func` — исходная функция
- `wrapper` — новая функция-обёртка
- `return wrapper` — декоратор возвращает новую версию функции

## Синтаксис с @
```python
def decorator(func):
    def wrapper():
        print("before")
        func()
        print("after")
    return wrapper

@decorator
def say_hello():
    print("hello")
```

Это почти то же самое, что:
```python
say_hello = decorator(say_hello)
```

## Если у функции есть аргументы
Тогда в `wrapper` обычно используют `*args, **kwargs`.

```python
def decorator(func):
    def wrapper(*args, **kwargs):
        print("before")
        result = func(*args, **kwargs)
        print("after")
        return result
    return wrapper
```

## Что важно
- декоратор возвращает функцию,
- `wrapper` вызывает исходную функцию,
- если исходная функция что-то возвращает, `wrapper` тоже должен вернуть результат.
