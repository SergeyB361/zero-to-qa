# Неделя 8, День 5 — GitHub workflows

## Что здесь имеется в виду
На этом дне фокус не на Git как локальном инструменте, а на рабочем процессе вокруг GitHub:
- issues
- pull requests
- README
- базовое оформление репозитория

## Почему это важно
Хороший репозиторий для QA — это не просто код. Он должен быть:
- понятным
- воспроизводимым
- читаемым для другого человека

## Что обычно полезно
- понятный `README`
- структура проекта
- описание запуска
- осмысленные названия pull request
- ссылки на issues, если они есть

## Практические правила
- думай о репозитории как о рабочем продукте, а не как о личной папке
- если другой человек не понимает, как запустить проект, значит оформление ещё слабое
- README должен помогать, а не быть пустой формальностью
