# Неделя 8, День 4 — Git для QA

## Зачем Git нужен QA
Git нужен не только разработчикам. Для QA он важен, чтобы:
- хранить автотесты и утилиты
- отслеживать историю изменений
- работать через ветки и pull request
- безопасно синхронизировать работу между компьютерами

## Базовые команды
- `git status`
- `git add`
- `git commit`
- `git push`
- `git pull`

## Что такое ветка
Ветка — это отдельная линия изменений. Обычно новая работа идёт не прямо в основной ветке, а в отдельной feature-ветке.

## Что такое pull request
Pull request — это способ предложить изменения на ревью и обсудить их перед слиянием.

## Практические правила
- коммиты должны быть осмысленными и небольшими
- перед началом работы полезно синхронизироваться с удалённым репозиторием
- не стоит смешивать в одном коммите несвязанные изменения
