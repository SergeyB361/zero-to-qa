# Неделя 4, День 2 — Методы

## Что такое методы
Методы — это функции внутри класса. Они описывают поведение объекта или самого класса.

## Три основных вида методов
- обычный метод: работает с объектом через `self`
- `@classmethod`: работает с классом через `cls`
- `@staticmethod`: логически относится к классу, но не использует ни `self`, ни `cls`

## Пример
```python
class User:
    total = 0

    def __init__(self, name):
        self.name = name
        User.total += 1

    def greet(self):
        return f"Hello, {self.name}"

    @classmethod
    def count(cls):
        return cls.total

    @staticmethod
    def is_valid_name(name):
        return len(name) >= 2
```

## Как вызываются методы
```python
user = User("Ann")
print(user.greet())
print(User.count())
print(User.is_valid_name("Ann"))
```

Важно понимать:
- `user.greet()` работает с конкретным объектом `user`
- `User.count()` работает с самим классом `User`
- `User.is_valid_name("Ann")` просто выполняет связанную с классом проверку

## `self` и `cls`
- `self` — конкретный объект
- `cls` — сам класс

Их не нужно передавать вручную при обычном вызове:
- `user.greet()` автоматически передаст объект в `self`
- `User.count()` автоматически передаст класс в `cls`

## Атрибут объекта и атрибут класса
В примере выше:
- `self.name` — атрибут конкретного объекта
- `User.total` / `cls.total` — атрибут класса, общий для всех объектов

Это важно: объект хранит свои данные отдельно, а класс может хранить общее состояние.

## Когда что использовать
- обычный метод: читает или меняет состояние объекта
- `@classmethod`: фабрики, счётчики, работа с данными класса
- `@staticmethod`: утилиты и проверки, которые логически относятся к классу

## Практические правила
- Если метод использует данные объекта, нужен обычный метод.
- Если метод должен работать с общим состоянием класса, нужен `@classmethod`.
- Если функция просто тематически относится к классу, но не зависит от объекта и класса, подойдёт `@staticmethod`.
- Не используй `@staticmethod` просто потому, что так короче: сначала проверь, не лучше ли обычная функция вне класса.
