# Неделя 4, День 4 — Инкапсуляция

## Что это значит
Инкапсуляция помогает управлять доступом к данным объекта и прятать внутренние детали реализации.

Идея простая:
- снаружи объект должен иметь удобный и понятный интерфейс
- внутри объект может хранить данные так, как ему удобно

## Почему это полезно
Инкапсуляция нужна, когда:
- данные надо валидировать
- нельзя допускать некорректные значения
- хочется поменять внутреннюю реализацию, не ломая внешний код

## Соглашения в Python
- `_name` — внутренний атрибут по соглашению
- `__name` — name mangling, усложняет прямой доступ извне
- `@property` — даёт удобный интерфейс чтения и записи

Важно: в Python инкапсуляция мягкая. Язык не делает поля по-настоящему приватными как в некоторых других языках, но даёт инструменты и соглашения.

## Пример
```python
class User:
    def __init__(self, name, age):
        self.name = name
        self._age = age

    @property
    def age(self):
        return self._age

    @age.setter
    def age(self, value):
        if value < 0:
            raise ValueError("age must be >= 0")
        self._age = value
```

## Как это работает
```python
user = User("Ann", 25)
print(user.age)
user.age = 30
```

Снаружи код выглядит так, будто `age` — обычный атрибут. Но внутри у нас есть контроль через getter и setter.

## Когда использовать `@property`
`@property` полезен, когда:
- нужна валидация
- нужно вычислять значение на лету
- атрибут должен быть только для чтения

## Практические правила
- Не прячь всё подряд: инкапсуляция нужна там, где есть валидация или важные ограничения.
- Через `@property` можно оставить удобный внешний API.
- Если атрибут должен быть только для чтения, setter можно не писать.
- `_name` — это сигнал: "не трогай без необходимости".
