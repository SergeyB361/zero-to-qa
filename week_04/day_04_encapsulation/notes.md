# Неделя 4, День 4 — Инкапсуляция

## Что это значит
Инкапсуляция помогает управлять доступом к данным объекта и прятать внутренние детали реализации.

## Соглашения в Python
- `_name` — внутренний атрибут по соглашению
- `__name` — name mangling, усложняет прямой доступ извне
- `@property` — даёт удобный интерфейс чтения и записи

## Пример
```python
class User:
    def __init__(self, name, age):
        self.name = name
        self._age = age

    @property
    def age(self):
        return self._age

    @age.setter
    def age(self, value):
        if value < 0:
            raise ValueError("age must be >= 0")
        self._age = value
```

## Практические правила
- Не прячь всё подряд: инкапсуляция нужна там, где есть валидация или важные ограничения.
- Через `@property` можно оставить удобный внешний API.
- Если атрибут должен быть только для чтения, setter можно не писать.
