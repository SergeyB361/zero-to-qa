# Неделя 4, День 5 — ООП на практике

## Зачем это нужно в QA
Классы помогают группировать данные и поведение, чтобы код тестов и утилит был понятнее.

## Где это полезно
- модели пользователя, заказа, окружения
- helper-классы для API и UI
- представление тест-кейсов и баг-репортов

## Идея проектирования
Перед созданием класса ответь на три вопроса:
1. Какие данные хранит объект?
2. Какие действия объект умеет делать?
3. Что должно быть публичным API?

## Пример
```python
class TestCase:
    def __init__(self, title, status="draft"):
        self.title = title
        self.status = status

    def mark_passed(self):
        self.status = "passed"
```
