# Неделя 4, День 6 — Модули и пакеты

## Что такое модуль
Модуль — это `.py`-файл с кодом, который можно импортировать.

## Что такое пакет
Пакет — это папка с Python-модулями. Часто внутри есть `__init__.py`.

## Базовые импорты
```python
import math
from pathlib import Path
from my_module import run
```

## Как Python ищет модули
Когда ты пишешь `import helper`, Python ищет модуль:
- в текущей папке запуска
- в стандартной библиотеке
- в путях из `sys.path`

Для этой практики это важно: если ты создашь `helper.py` рядом с `practice.py`, то импорт сработает только если запускать файл из той же папки или если эта папка попадает в путь поиска модулей.

Надежный вариант для этого дня:
1. открой терминал в папке `day_06_modules_packages`
2. запускай `python practice.py`

Если запускать файл из другой директории, локальный `import helper` может не сработать, даже если `helper.py` существует.

## Зачем это нужно
- разделять код по файлам
- переиспользовать функции и классы
- держать проект в понятной структуре

## Практические правила
- один модуль — одна понятная зона ответственности
- не дублируй код, выноси его в отдельные модули
- избегай циклических импортов
- для локальных импортов сначала проверь, из какой папки реально запускается файл

## Минимальная схема для задания 3
Структура должна выглядеть так:

```text
day_06_modules_packages/
  practice.py
  helper.py
```

Пример содержимого `helper.py`:

```python
def say_hi(name):
    return f"Hi, {name}"
```

Пример импорта в `practice.py`:

```python
from helper import say_hi

print(say_hi("Sergey"))
```

## Практика
Открой `practice.py` и потренируй импорты и разбиение кода на файлы.
