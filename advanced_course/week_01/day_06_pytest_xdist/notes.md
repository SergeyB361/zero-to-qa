# PyTest: продвинутый уровень, День 6 — pytest-xdist

## Зачем это нужно
`pytest-xdist` ускоряет suite за счёт параллельного запуска.
Но ускорение полезно только тогда, когда тесты уже достаточно изолированы.

## Базовая идея
Команда вида:
- `pytest -n auto`

запускает тесты на нескольких worker-процессах.

Это значит:
- порядок запуска больше не гарантирован
- shared state становится опаснее
- один общий файл или общий порт быстро ломают suite

## Что должно быть xdist-safe
- независимые тесты
- уникальные временные пути
- отсутствие глобального изменяемого состояния
- отсутствие зависимости от порядка тестов

## Что чаще всего ломается
- запись в общий файл
- общая БД без очистки
- фиксированный TCP-порт
- глобальные счётчики
- session fixture с изменяемым объектом

## Практическое правило
Сначала делай тесты чистыми и изолированными.
Потом включай параллельность.

Если suite без xdist уже нестабилен, xdist только ускорит хаос.

## Полезные команды
- `pytest -n auto`
- `pytest -n 4`
- `pytest -n auto -m smoke`
