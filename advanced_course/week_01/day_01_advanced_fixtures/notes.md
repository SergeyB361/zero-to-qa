# PyTest: продвинутый уровень, День 1 — Advanced fixtures

## Зачем это нужно
На базовом уровне fixture обычно просто возвращает данные. На продвинутом уровне она становится инструментом сборки тестовой среды:
- подготавливает зависимости
- управляет teardown
- комбинируется с другими fixture
- уменьшает copy-paste

## Что важно понять
Fixture — это не «магия pytest», а способ описать reusable setup.

Полезные паттерны:
- composition: одна fixture использует другую
- yield fixture: setup до `yield`, teardown после
- factory fixture: fixture возвращает функцию-конструктор
- scope: сколько раз создаётся объект (`function`, `class`, `module`, `session`)
- autouse: автоматическое применение без явного аргумента в тесте

## Composition
Это самый здоровый способ собирать тестовые данные.

Пример идеи:
- `user_data` создаёт обычного пользователя
- `admin_user` берёт `user_data` и меняет роль

Такой подход лучше, чем копировать словари по всем тестам.

## Yield fixture
Используй `yield`, когда нужен teardown:
- открыть файл / закрыть файл
- создать временный ресурс / удалить его
- подготовить состояние / очистить состояние

Модель простая:
1. код до `yield` — setup
2. значение из `yield` идёт в тест
3. код после `yield` — teardown

## Factory fixture
Если тестам нужны похожие, но немного разные объекты, удобнее возвращать функцию.

Это полезно для:
- пользователей с разными ролями
- заказов с разными статусами
- payload с разными полями

## Scope
`function`:
- самый безопасный вариант
- новый объект на каждый тест

`module` / `session`:
- быстрее
- но выше риск shared state и протечек между тестами

Практическое правило:
- сначала выбирай `function`
- расширяй scope только когда понимаешь, почему это безопасно

## Antipatterns
Плохие признаки:
- fixture делает слишком много
- fixture скрывает важную бизнес-логику
- один gigantic fixture используется в половине проекта
- session fixture хранит изменяемое состояние

Хорошая fixture обычно:
- короткая
- предсказуемая
- локальная по смыслу
- не прячет критичные детали от теста

## Что запомнить
- fixture — это reusable setup, а не место для сложной магии
- composition почти всегда лучше copy-paste
- `yield` нужен там, где есть teardown
- расширенный scope ускоряет, но повышает риск нестабильности
