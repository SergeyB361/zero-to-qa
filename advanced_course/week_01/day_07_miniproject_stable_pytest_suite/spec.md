# ТЗ: Мини-проект — Stable PyTest suite

## Исходный файл
Используй:
- qa_service.py

## Цель проекта
Собрать мини-набор PyTest тестов, который показывает fixture, parametrization, mocking и anti-flaky подходы.

## Deliverables
- tests/test_qa_service.py с pytest-тестами
- минимум одна fixture
- минимум один parametrized test
- минимум один тест с Mock или monkeypatch
- короткая инструкция запуска

## Обязательные требования
- suite запускается локально одной командой pytest
- не использовать sleep как основной способ стабилизации
- каждый тест имеет понятный assert и читаемое имя
- минимум один edge или error path

## Формат сдачи
- файлы проекта в папке дня
- показать вывод pytest
- объяснить anti-flaky решения

## Рубрика оценки (10 баллов)
- 2 балла — структура suite и имена тестов
- 2 балла — корректное использование fixtures и parametrization
- 2 балла — mocking сделан по делу
- 2 балла — flaky-risk снижен и объяснён
- 2 балла — suite даёт понятную диагностику

## Что особенно проверяется
- решение должно быть структурным, а не набором случайных заметок
- каждое принятое решение должно быть объяснимо через риск, цель или качество
- материал должен быть пригоден для ревью и дальнейшего расширения
