# PyTest: продвинутый уровень, День 7 — Мини-проект: устойчивый PyTest suite

## Цель
На этой неделе важно не просто знать отдельные инструменты, а собрать их в один аккуратный suite.

В мини-проекте ты тренируешь:
- fixture composition
- factory fixture
- parametrization
- markers
- mocking внешних зависимостей
- xdist-safe подход к файлам и состоянию

## Что будет считаться хорошим результатом
Хороший suite здесь:
- читаемый
- изолированный
- не зависит от порядка запуска
- не пишет в общие файлы
- не использует sleep без причины
- не тянет внешнюю сеть в unit-тесты

## Как лучше идти
1. Сначала прочитай `qa_service.py`.
2. Раздели функции на pure и impure.
3. Для pure logic используй параметризацию.
4. Для внешних зависимостей используй Mock.
5. Для файлов используй `tmp_path`.
6. Добавь markers там, где это реально помогает.

## Практический чек-лист
Перед сдачей проекта проверь:
- fixture не скрывают важную логику
- ids в параметризации читаемые
- mock проверяет контракт вызова, а не всё подряд
- тесты не делят общий файл или общий mutable state
- suite можно безопасно запускать с xdist
