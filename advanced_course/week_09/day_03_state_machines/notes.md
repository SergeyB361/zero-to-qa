# Test Design и Requirements Engineering, День 3 — State machines

## Зачем это нужно
- State machines помогают думать о системе как о наборе состояний и переходов.
- Это особенно полезно для заказов, workflow, сессий и approval процессов.
- Модель состояний делает forbidden transitions видимыми сразу.

## Что важно понять
- Есть состояния, события и переходы.
- Для каждого состояния важно понимать, какие события допустимы.
- Negative testing здесь строится вокруг forbidden transitions.

## Частые ошибки
- Проверять только happy path переходы.
- Не фиксировать, что происходит при недопустимом событии.
- Игнорировать начальное и terminal states.
