# Test Design и Requirements Engineering, День 2 — Decision tables

## Зачем это нужно
- Decision table помогает системно покрывать комбинации условий и действий.
- Это один из самых практичных инструментов тест-дизайна для бизнес-правил.
- Он особенно полезен там, где много флагов, ролей, скидок и статусов.

## Что важно понять
- В таблице есть условия, действия и правила-комбинации.
- Таблица помогает увидеть пропуски и лишние дубли сценариев.
- Она полезна не только для тестов, но и для уточнения требований.

## Частые ошибки
- Не фиксировать ожидаемое действие для каждой комбинации.
- Пропускать одну из ветвей true/false.
- Пытаться засунуть в одну таблицу несвязанные бизнес-правила.

## Практика дня
Открой practice.py и построй decision table для правил скидки или доступа.
