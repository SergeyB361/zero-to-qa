# ТЗ: Пакет тест-дизайна по набору требований

## Исходный файл
Используй:
- requirements.md

## Что нужно подготовить
1. EARS-переформулировки для расплывчатых требований
2. decision table для бизнес-правила
3. state machine для жизненного цикла сущности
4. BVA и equivalence partitioning для числовых и строковых полей
5. pairwise ideas для нескольких параметров
6. основной и альтернативный user flow

## Обязательные требования
- все артефакты должны ссылаться на конкретные требования
- требования должны быть переведены в тестовые идеи, а не просто переписаны
- должен быть виден reasoning, почему выбраны именно эти техники
