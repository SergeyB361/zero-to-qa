# Test Design и Requirements Engineering, День 6 — Use cases и user flows

## Зачем это нужно
- Use cases и user flows помогают видеть сценарий глазами пользователя, а не только через поля и статусы.
- Они полезны для выделения основного потока и альтернативных ветвей.
- Это хорошая связка между требованиями и UI или e2e тестами.

## Что важно понять
- Есть основной поток, альтернативные потоки и error flows.
- Flow должен быть достаточно подробным для тест-дизайна, но без лишней воды.
- Хороший flow помогает выявить, где нужны state transitions и decision tables.

## Частые ошибки
- Описывать только happy path.
- Смешивать несколько user goals в одном use case.
- Не фиксировать, где именно поток отклоняется.
