# Test Design и Requirements Engineering, День 5 — Pairwise testing

## Зачем это нужно
- Когда параметров много, полное покрытие комбинаций быстро становится слишком дорогим.
- Pairwise testing помогает сократить число сценариев, сохранив хорошее покрытие пар значений.
- Это practical компромисс между хаосом и brute force.

## Что важно понять
- Pairwise не даёт полного покрытия всех комбинаций.
- Он покрывает пары значений между параметрами.
- Метод особенно полезен для конфигов, ролей, браузеров, тарифов и флагов.

## Частые ошибки
- Считать pairwise заменой всей стратегии тестирования.
- Применять pairwise там, где одна конкретная тройка условий критична.
- Не документировать, какие комбинации были сокращены осознанно.
