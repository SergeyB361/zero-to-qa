# Test Design и Requirements Engineering, День 4 — BVA и Equivalence Partitioning

## Зачем это нужно
- Boundary Value Analysis и Equivalence Partitioning дают системный способ выбирать тестовые значения.
- Они помогают уменьшить количество тестов без потери смысла.
- Это базовые, но очень сильные техники для любого QA.

## Что важно понять
- Equivalence classes группируют значения, которые система обрабатывает одинаково.
- Boundary analysis фокусируется на границах этих классов.
- Вместе они уменьшают хаос в выборе тестовых данных.

## Частые ошибки
- Выбирать только один happy value.
- Проверять границу, но не брать значение сразу рядом с ней.
- Смешивать разные классы эквивалентности в одной идее теста.
