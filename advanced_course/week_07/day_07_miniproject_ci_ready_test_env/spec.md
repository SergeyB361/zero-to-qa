# ТЗ: Мини-проект — CI-ready test environment

## Исходный файл
Используй:
- docker-compose.yml

## Что нужно подготовить
1. локальную compose-среду для app и db
2. схему CI pipeline
3. quality gates
4. список observability сигналов
5. набор артефактов, сохраняемых при падении

## Обязательные требования
- среда должна описываться как код
- pipeline должен разделять быстрые и тяжёлые проверки
- quality gates должны быть измеримыми
- артефакты должны быть пригодны для расследования после CI
