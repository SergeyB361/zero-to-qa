# Infrastructure for QA, День 2 — docker-compose basics

## Зачем это нужно
- Тестовый проект редко живёт с одним контейнером — обычно нужны app, db и mock service.
- Compose позволяет описать multi-service окружение как код.
- Это удобный способ поднять локальную интеграционную среду для QA.

## Что важно понять
- Compose описывает сервисы, зависимости, переменные и сети.
- QA часто использует compose для локальной среды приложения плюс БД.
- Описание окружения должно быть воспроизводимым и читаемым.

## Частые ошибки
- Случайные локальные правки compose без фиксации в git.
- Отсутствие разделения между dev и test env.
- Секреты внутри compose файла без контроля.

## Практика дня
Открой practice.py и набросай compose-состав для app, db и mock service.
