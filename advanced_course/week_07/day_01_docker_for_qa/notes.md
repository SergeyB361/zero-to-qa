# Infrastructure for QA, День 1 — Docker for QA

## Зачем это нужно
- Docker делает тестовое окружение воспроизводимым.
- Он уменьшает зависимость от локальной машины инженера.
- Для QA это способ стабильно поднимать сервисы и утилиты рядом с тестами.

## Что важно понять
- Контейнер — это изолированный процесс с предсказуемым окружением.
- Docker полезен для сервисов, БД и тестовых утилит.
- Нужно понимать image, container, volume и port mapping.

## Частые ошибки
- Считать Docker заменой понимания окружения.
- Хранить секреты прямо в image.
- Использовать образы без контроля версий.
