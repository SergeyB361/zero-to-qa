# Infrastructure for QA, День 3 — CI pipelines

## Зачем это нужно
- CI pipeline превращает тесты из локальной привычки в инженерный процесс.
- Пайплайн должен проверять код последовательно и воспроизводимо.
- Для QA важно проектировать шаги pipeline под разные типы тестов.

## Что важно понять
- Обычно pipeline делится на setup, test, reports и artifacts.
- Matrix builds и caching могут ускорять прогон.
- Не все тесты должны запускаться на каждый push.

## Частые ошибки
- Один giant pipeline step на всё подряд.
- Нет разделения smoke и full regression.
- Нет сохранения артефактов при падении.

Открой practice.py и предложи базовую структуру CI pipeline для QA проекта.
