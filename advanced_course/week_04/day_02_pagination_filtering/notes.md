# API: reliability и integration, День 2 — Pagination и filtering

## Зачем это нужно
API с коллекциями почти всегда использует pagination, sorting и filtering. Ошибки в этих механиках незаметны на маленьких наборах, но быстро ломают real usage.

## Что важно понять
При pagination важно проверять:
- limit и page или offset
- отсутствие дубликатов между страницами
- корректное количество элементов
- детерминированный порядок

При filtering важно проверять:
- фильтр действительно сужает выборку
- поля фильтра поддерживают ожидаемые значения
- комбинации фильтров не дают неожиданный шум

## Частые ошибки
- проверять только первую страницу
- не сравнивать объединение страниц с полным набором
- игнорировать порядок результатов
- считать фильтр корректным без проверки содержимого

## Что запомнить
- pagination и filtering надо проверять вместе и отдельно
- порядок выдачи влияет на стабильность тестов
- дубликаты и пропуски между страницами — критический дефект
- фильтр должен проверяться по содержимому, а не только по count
