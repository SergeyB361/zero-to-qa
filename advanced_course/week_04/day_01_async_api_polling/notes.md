# API: reliability и integration, День 1 — Async API и polling

## Зачем это нужно
- Многие API возвращают не готовый результат, а job id или промежуточный статус.
- Тест должен уметь ждать завершения по правилу, а не через случайный sleep.
- Eventual consistency часто является частью дизайна, а не дефектом.

## Что важно понять
- Polling helper должен иметь condition, timeout и max attempts.
- Проверять нужно и промежуточные статусы, и финальный результат.
- Timeout и never-complete сценарии обязательны.

## Частые ошибки
- Бесконечный polling без лимита.
- Sleep без причины вместо явного условия.
- Проверка только happy path.

Открой practice.py и собери helper для polling по списку статусов.
