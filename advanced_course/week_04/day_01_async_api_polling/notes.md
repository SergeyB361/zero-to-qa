# API: reliability и integration, День 1 — Async API и polling

## Зачем это нужно
Некоторые API не возвращают финальный результат сразу. Они запускают задачу и дают статус или job id. Тест тогда должен не просто сделать один запрос, а дождаться достижения terminal state.

## Что важно понять
Polling — это повторная проверка статуса с паузой и limit на число попыток.

Ключевые вопросы:
- какие статусы считаются промежуточными
- какие статусы terminal
- сколько попыток допустимо
- что считать timeout

## Хороший polling helper
Полезный helper обычно умеет:
- принимать список pending statuses
- завершаться на terminal success
- быстро падать на terminal failure
- давать понятную диагностику при timeout

## Частые ошибки
- бесконечный цикл без лимита
- игнорирование fail state
- слишком короткий или слишком длинный polling interval
- отсутствие диагностики по последнему статусу

## Что запомнить
- polling обязан иметь deadline или max attempts
- pending и terminal statuses должны быть заданы явно
- failure state надо обрабатывать сразу
- диагностика при timeout так же важна, как и success path
