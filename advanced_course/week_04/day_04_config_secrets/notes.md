# API: reliability и integration, День 4 — Config и secrets

## Зачем это нужно
- Зрелый фреймворк не должен хранить URL и токены хаотично по коду.
- Ошибки в secrets management быстро приводят к утечкам и нестабильным окружениям.
- Config и secrets требуют разного уровня защиты.

## Что важно понять
- Static config, environment config и secrets нужно разделять.
- Secrets не должны попадать в git, логи и stack traces.
- Безопасные defaults допустимы только для не-чувствительных полей.

## Частые ошибки
- Хардкодить токен в код.
- Логировать секрет целиком.
- Смешивать base URL и секреты в одном случайном модуле.
