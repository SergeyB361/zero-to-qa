# Test Data Engineering, День 1 — Factories

## Зачем это нужно
- Factories уменьшают copy-paste тестовых данных.
- Они позволяют централизованно менять форму сущности.
- Они ускоряют создание вариаций happy path и edge cases.

## Что важно понять
- Factory — это функция или класс для создания тестового объекта по правилам.
- Хорошая factory имеет осмысленные значения по умолчанию.
- Factory должна быть предсказуемой и без скрытой случайности.

## Частые ошибки
- Один giant factory на все сущности сразу.
- Случайные значения без контроля seed.
- Default значения, которые не похожи на реальный домен.

## Практика дня
Открой practice.py и собери reusable factory для пользователя, заказа и тикета.
