# Test Data Engineering, День 4 — Property-based testing

## Зачем это нужно
- Property-based testing проверяет свойства на множестве входов, а не только отдельные примеры.
- Эта техника особенно сильна для pure logic и преобразований данных.
- Она помогает найти неожиданные входы и минимальный failing case.

## Что важно понять
- Property — это инвариант, который должен выполняться для широкого класса входов.
- Это не просто random testing: генерация идёт по правилам.
- Лучше всего техника работает там, где свойство можно чётко сформулировать.

## Частые ошибки
- Слишком расплывчатое свойство.
- Попытка применять подход к сложному UI-flow.
- Подмена свойства просто списком обычных примеров.
