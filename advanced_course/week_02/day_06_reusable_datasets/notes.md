# Test Data Engineering, День 6 — Reusable datasets

## Зачем это нужно
Некоторые тестовые наборы повторяются постоянно: роли, статусы, тарифы, типы устройств, локали. Если каждый тест описывает их заново, suite быстро расходится по смыслу.

Reusable dataset нужен как компактный каталог доменных наборов.

## Что важно понять
Reusable dataset — это не свалка всех возможных значений.
Он должен:
- быть небольшим
- отражать реальные доменные категории
- использоваться в нескольких местах

Хороший dataset отвечает на вопрос: какой минимальный набор значений реально нужен тестам?

## Где это особенно полезно
- user roles
- order statuses
- browser matrix
- locale matrix
- обязательные негативные комбинации

## Разделение по смыслу
Удобно держать отдельно:
- happy datasets
- restricted или blocked datasets
- edge datasets

Тогда тест читает не список сырых значений, а понятное доменное имя.

## Antipatterns
- огромный список значений без группировки
- набор называется общо и ничего не объясняет
- dataset дублирует данные factory без причины
- mutable dataset меняется прямо в тесте

## Что запомнить
- reusable dataset — это доменный каталог, а не склад всего подряд
- наборы должны быть маленькими и говорящими
- группируй значения по бизнес-смыслу
- не смешивай reusable и dynamic data без причины
