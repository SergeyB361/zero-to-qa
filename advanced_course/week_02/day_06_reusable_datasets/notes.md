# Test Data Engineering, День 6 — Reusable datasets

## Зачем это нужно
- Reusable datasets позволяют использовать одни и те же доменные сценарии в unit, API и UI слоях.
- Они делают тестовые наборы согласованными между командами.
- Они уменьшают хаос в данных и упрощают поддержку.

## Что важно понять
- Хороший dataset имеет имя, смысл и предсказуемое содержимое.
- Dataset должен отражать конкретный доменный сценарий, а не просто набор полей.
- Dataset не должен превращаться в кладбище старых кейсов.

## Частые ошибки
- Случайные значения внутри reusable dataset.
- Непонятные имена вроде data1/data2.
- Один огромный каталог со всем подряд без структуры.

## Практика дня
Открой practice.py и собери небольшой каталог reusable datasets для пользователей и заказов.
