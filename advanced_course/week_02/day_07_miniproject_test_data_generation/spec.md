# ТЗ: Мини-проект — генерация и проверка тестовых данных

## Исходный файл
Используй:
- data_targets.py

## Цель проекта
Построить toolkit для генерации тестовых данных и проверки их целостности без giant JSON и хаотичных overrides.

## Deliverables
- factories.py с минимум двумя factory-функциями
- builders.py с минимум одним builder
- datasets.py с reusable наборами
- rules.py с инвариантами или валидаторами
- опционально demo.py

## Обязательные требования
- default values валидны и предсказуемы
- минимум две factory для разных сущностей
- минимум один builder с chainable методами
- минимум два reusable dataset и два edge набора
- минимум три явные проверки-инварианта

## Формат сдачи
- файлы решения в папке проекта
- показать один happy dataset и один edge dataset
- объяснить логику разделения данных

## Рубрика оценки (10 баллов)
- 2 балла — данные читаемы и доменно понятны
- 2 балла — factories и builder реально уменьшают copy-paste
- 2 балла — datasets разделены по смыслу
- 2 балла — инварианты проверяют качество данных
- 2 балла — решение легко расширять

## Что особенно проверяется
- решение должно быть структурным, а не набором случайных заметок
- каждое принятое решение должно быть объяснимо через риск, цель или качество
- материал должен быть пригоден для ревью и дальнейшего расширения
