# Test Data Engineering, День 2 — Builder pattern

## Зачем это нужно
- Builder удобен для сложных объектов с большим числом полей.
- Он делает пошаговую сборку объекта читаемой.
- Он полезен, когда сигнатура factory становится перегруженной.

## Что важно понять
- Builder обычно имеет стартовое состояние, chainable методы и build().
- Builder хорошо подходит для payload с optional полями.
- Builder не должен скрывать побочные эффекты внутри build().

## Частые ошибки
- Builder ради объекта из двух полей.
- Случайные значения внутри build().
- Builder, который меняет глобальное состояние.

## Практика дня
Открой practice.py и собери TicketBuilder или OrderBuilder с chainable методами.
