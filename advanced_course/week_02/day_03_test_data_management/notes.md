# Test Data Engineering, День 3 — Test data management

## Зачем это нужно
- Создать данные мало — их нужно ещё хранить и поддерживать.
- Большой проект требует разделения на static, generated и environment-specific data.
- Управление данными напрямую влияет на стабильность тестов.

## Что важно понять
- Static data хорошо подходит для ролей, статусов и тарифов.
- Generated data нужна для уникальных email, id и временных сущностей.
- Environment-specific data не должна течь между окружениями и git.

## Частые ошибки
- Один giant JSON на всё.
- Тесты меняют общий dataset.
- Данные завязаны на текущую дату без контроля.

## Практика дня
Открой practice.py и разложи данные по reusable и dynamic слоям.
