# UI: продвинутый уровень, День 6 — Debugging UI failures

## Зачем это нужно
- UI failure редко объясняется одной строкой assert.
- Для расследования нужны trace, screenshot, console logs и network context.
- Без дисциплины диагностика UI-падений превращается в угадывание.

## Что важно понять
- Нужно собирать screenshot, page source, trace или video по смыслу.
- Важен не только артефакт, но и его привязка к тесту и шагу.
- Debugging должен быть воспроизводимым, а не разовым ритуалом.

## Частые ошибки
- Скриншот без контекста теста и шага.
- Артефакты без именования и структуры.
- Нет логов консоли или сети при расследовании UI-bug.

## Практика дня
Открой practice.py и собери минимальный набор артефактов для расследования UI падения.
