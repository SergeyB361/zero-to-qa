# UI: продвинутый уровень, День 1 — Auth state

## Зачем это нужно
- Повторный логин в каждом UI-тесте замедляет suite и повышает нестабильность.
- Storage state и session reuse экономят время и уменьшают шум.
- Auth state важно отделять по ролям и окружениям.

## Что важно понять
- Session reuse полезен, если состояние стабильно и изолировано.
- Нужно понимать, когда storage state устаревает.
- Auth state нельзя путать с permanent shared state.

## Частые ошибки
- Один auth state для всех ролей.
- Reuse устаревшей сессии без проверки.
- Потеря изоляции между тестами.
