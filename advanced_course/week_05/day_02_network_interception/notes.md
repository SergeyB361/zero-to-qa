# UI: продвинутый уровень, День 2 — Network interception

## Зачем это нужно
- Network interception делает UI-тесты детерминированнее.
- Она полезна для редких ответов, ошибок и частичной изоляции от бэкенда.
- Через interception удобно стабилизировать UI на уровне сетевого слоя.

## Что важно понять
- Можно подменять ответ, блокировать запрос и проверять сам факт вызова.
- Interception полезен для flaky и редких error-сценариев.
- Он не должен полностью заменить реальные end-to-end smoke.

## Частые ошибки
- Mock всего приложения без реальной интеграции.
- Слишком хрупкая логика по URL-подстрокам.
- Подмена ответов, не похожих на реальный контракт.
