# UI: продвинутый уровень, День 5 — Page Object architecture

## Зачем это нужно
- POM нужен для отделения тестового сценария от деталей страницы.
- Без архитектуры UI-suite быстро превращается в copy-paste действий.
- Компоненты и reusable flows делают suite поддерживаемым.

## Что важно понять
- Page object должен скрывать механику, но не скрывать бизнес-логику теста.
- Общие элементы удобно выносить в page components.
- Слишком умный BasePage почти всегда вреден.

## Частые ошибки
- God object на все страницы сразу.
- Бизнес-assert внутри page object.
- BasePage с десятками неиспользуемых методов.

## Практика дня
Открой practice.py и разложи, что должно жить в Page Object, а что в самом тесте.
