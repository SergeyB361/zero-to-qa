# UI: продвинутый уровень, День 4 — Locator strategy

## Зачем это нужно
- Большая часть UI-flakiness приходит из плохих локаторов.
- Стратегия локаторов важнее конкретного синтаксиса CSS или XPath.
- Хороший локатор должен переживать нормальные изменения верстки.

## Что важно понять
- Сильнее всего обычно role, label, text и test-id.
- Слабее всего длинные CSS/XPath цепочки по DOM-структуре.
- Нужно выбирать локатор по смыслу взаимодействия, а не по случайному удобству.

## Частые ошибки
- Локаторы по nth-child и глубокой DOM-структуре.
- Зависимость от случайных class name.
- Использование текста там, где UI часто меняется.
