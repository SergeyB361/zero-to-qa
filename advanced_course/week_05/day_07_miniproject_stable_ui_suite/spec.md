# ТЗ: Мини-проект — Stable UI suite

## Исходный файл
Используй:
- page_model.py

## Цель проекта
Спроектировать устойчивый UI suite с осмысленной locator strategy, waiting conditions и Page Object разделением.

## Deliverables
- минимум один Page Object и один reusable flow
- описание locator strategy
- правила waits для ключевых шагов
- набор anti-flaky решений

## Обязательные требования
- Page Object не прячет business asserts
- локаторы ранжированы по устойчивости
- есть минимум один пример правильного wait
- описан подход к debug UI failures

## Формат сдачи
- файлы page objects и описание flows
- короткий пример одного UI теста
- список решений против flaky risk

## Рубрика оценки (10 баллов)
- 2 балла — Page Object отделён от логики теста
- 2 балла — locator strategy объяснима и устойчива
- 2 балла — waits выбраны по событию, а не по sleep
- 2 балла — anti-flaky решения практичны
- 2 балла — suite можно расширять без copy-paste

## Что особенно проверяется
- решение должно быть структурным, а не набором случайных заметок
- каждое принятое решение должно быть объяснимо через риск, цель или качество
- материал должен быть пригоден для ревью и дальнейшего расширения
