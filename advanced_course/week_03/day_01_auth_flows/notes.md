# API: продвинутый уровень, День 1 — Auth flows

## Зачем это нужно
- Реальные API редко ограничиваются одним static token.
- QA должен понимать access token, refresh token, OAuth и JWT claims.
- Ошибки в авторизации часто критичнее happy path.

## Что важно понять
- Нужно разделять получение токена, хранение токена и построение auth headers.
- Negative auth scenarios так же важны, как и успешный вход.
- Секреты и реальные токены нельзя тащить в git и логи.

## Частые ошибки
- Хардкодить реальные секреты.
- Смешивать получение токена и бизнес-assert в одном тесте.
- Проверять только статус 200 без проверки claims и прав.

## Практика дня
Открой practice.py и отработай access token, refresh token и негативные auth сценарии.
