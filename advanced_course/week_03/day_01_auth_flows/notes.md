# API: продвинутый уровень, День 1 — Auth flows

## Зачем это нужно
На практике API-тесты редко живут только вокруг happy path без авторизации. Нужно понимать, как сервис выдаёт, обновляет и проверяет токены.

## Что важно понять
Минимальный auth flow обычно включает:
- получение access token
- использование токена в запросах
- обновление access token через refresh token
- негативные кейсы: expired, invalid, missing token

Важно различать:
- authentication: кто ты
- authorization: что тебе разрешено

## Где чаще всего ломается
- истёкший токен обрабатывается как 500
- refresh работает без нужной валидации
- роли и scope проверяются неполно
- один и тот же token reuse ломает stateful сценарий

## Как тестировать
Разделяй проверки на уровни:
- valid auth
- invalid и expired auth
- role-based access
- token refresh and reuse

## Что запомнить
- auth flow всегда требует позитивных и негативных сценариев
- access и refresh token надо тестировать отдельно
- 401 и 403 — это разные ошибки
- role и scope должны проверяться явно

Открой practice.py и отработай access token, refresh token и негативные auth сценарии.
