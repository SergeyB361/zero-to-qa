# API: продвинутый уровень, День 6 — Mock servers

## Зачем это нужно
Не все зависимости доступны, стабильны и дешевы для запуска в тестах. Mock server позволяет контролировать внешний сервис и воспроизводить нужные ответы.

## Что важно понять
Mock server полезен, когда нужно:
- воспроизвести редкую ошибку провайдера
- стабильно вернуть нужный payload
- не зависеть от real external system
- проверить reaction приложения на разные ответы

## Где он особенно уместен
- интеграции с платёжными провайдерами
- сторонние уведомления и webhooks
- rate limit, timeout и 5xx сценарии

## Где надо быть осторожным
Если мок слишком отрывается от реального провайдера, тест создаёт ложное чувство уверенности. Mock server нужен как controlled dependency, а не как полная замена контракта и интеграционного слоя.

## Частые ошибки
- мокать всё подряд без анализа реального риска
- возвращать слишком идеальный payload
- не тестировать timeout и error branches

## Что запомнить
- mock server помогает детерминировать внешнюю зависимость
- особенно полезны error и timeout сценарии
- мок должен оставаться близким к реальному контракту
- integration tests с реальным провайдером всё равно нужны на своём уровне
