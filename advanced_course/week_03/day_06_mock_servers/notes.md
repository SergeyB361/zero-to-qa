# API: продвинутый уровень, День 6 — Mock servers

## Зачем это нужно
- Mock server нужен для детерминированной проверки интеграции без реального внешнего сервиса.
- Он полезен, когда provider нестабилен или редкие ошибки трудно воспроизвести.
- Он помогает изолировать клиента от внешнего окружения.

## Что важно понять
- Mock server должен уметь отдавать success, error, timeout и разные сценарии одного endpoint.
- Mock server нужно сочетать с контрактными и smoke-проверками на реальной интеграции.
- Сценарии mock server должны отражать реальные риски, а не случайные фантазии.

## Частые ошибки
- Полностью заменить реальные интеграционные проверки mock-сервером.
- Моделировать только success case.
- Делать mock payload, который никогда не совпадал с реальным контрактом.
