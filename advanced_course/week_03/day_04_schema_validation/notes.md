# API: продвинутый уровень, День 4 — Schema validation

## Зачем это нужно
Когда payload сложнее пары полей, одних точечных assert уже мало. Schema validation позволяет быстро проверить required fields, типы, enum и вложенные структуры.

## Что важно понять
Schema validation не заменяет бизнес-проверки. Она отвечает на вопрос: payload вообще соответствует ожидаемой форме?

Полезно проверять:
- обязательные поля
- типы значений
- enum-поля
- минимальную структуру вложенных объектов

## Где это особенно выгодно
- API с большими JSON-ответами
- shared contracts между командами
- integration tests, где важен быстрый smoke формы ответа

## Частые ошибки
- проверять только happy payload
- не валидировать enum и required fields
- путать схему с полной бизнес-спецификацией

## Что запомнить
- schema validation быстро ловит drift формы ответа
- required fields, types и enum — минимальный baseline
- схема не заменяет предметную проверку значений
- negative payload тоже должен проверяться
