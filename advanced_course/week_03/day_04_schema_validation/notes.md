# API: продвинутый уровень, День 4 — Schema validation

## Зачем это нужно
- Schema validation помогает ловить drift структуры ответа.
- Она полезна там, где ad-hoc assert по отдельным полям уже не масштабируется.
- Schema защищает от тихих изменений в контракте.

## Что важно понять
- Schema проверяет required fields, типы, enum и вложенность.
- Schema не заменяет бизнес-assert — она отвечает только за структуру.
- Схему лучше переиспользовать, а не копировать по тестам.

## Частые ошибки
- Считать, что schema заменяет бизнес-логику.
- Проверять только одно поле вместо структуры.
- Дублировать схему вручную в каждом тесте.
