# ТЗ: Мини-проект — UI suite in CI

## Исходный файл
Используй:
- ci_plan.md

## Цель проекта
Подготовить план запуска UI suite в CI с browser matrix, артефактами и правилами стабилизации.

## Deliverables
- описание CI stages для UI suite
- browser matrix для smoke и regression
- список обязательных артефактов при падении
- подход к parallel execution и retries

## Обязательные требования
- есть разделение smoke и heavy suite
- артефакты включают минимум screenshot и trace/video/log
- описан browser matrix
- объяснено, какие сценарии safe для parallel run

## Формат сдачи
- ci_plan.md или эквивалентный файл
- пример набора артефактов для failed UI job
- объяснить выбранный browser scope

## Рубрика оценки (10 баллов)
- 2 балла — CI pipeline структурирован по скорости и сигналу
- 2 балла — browser matrix реалистичен
- 2 балла — артефакты помогают расследованию
- 2 балла — parallel strategy учитывает flaky risk
- 2 балла — решение выглядит как реальный CI plan

## Что особенно проверяется
- решение должно быть структурным, а не набором случайных заметок
- каждое принятое решение должно быть объяснимо через риск, цель или качество
- материал должен быть пригоден для ревью и дальнейшего расширения
