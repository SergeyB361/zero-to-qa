# UI: качество и масштабирование, День 6 — CI for UI tests

## Зачем это нужно
- UI suite в CI требует другой дисциплины, чем локальный запуск.
- Нужно управлять временем, артефактами, retries и селекцией запуска.
- Хорошая CI-стратегия делает UI запуск полезным, а не мучительным.

## Что важно понять
- В CI полезно разделять smoke и full regression.
- Артефакты падения должны сохраняться автоматически.
- Retry в CI допустим только как осознанный компромисс.

## Частые ошибки
- Гонять весь UI набор на каждый commit без фильтрации.
- Не сохранять traces, screenshots и video.
- Лечить все падения одним retry.

Открой practice.py и определи минимальный UI smoke для CI и набор обязательных артефактов.
