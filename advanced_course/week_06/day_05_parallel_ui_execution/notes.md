# UI: качество и масштабирование, День 5 — Parallel UI execution

## Зачем это нужно
- Параллельный запуск ускоряет UI-suite, но вскрывает shared state и гонки.
- Без изоляции данных и окружения parallel UI только умножает хаос.
- Нужно уметь оценивать, какие сценарии безопасны для concurrency.

## Что важно понять
- UI parallel execution требует изоляции accounts, files, ports и test data.
- Нельзя рассчитывать на стабильный порядок выполнения.
- Часть suite может идти параллельно, а часть — последовательно по бизнес-причине.

## Частые ошибки
- Один аккаунт на все тесты.
- Общий download или upload файл для всех worker.
- Слепое включение parallel без анализа конфликтов.

## Практика дня
Открой practice.py и отметь, какие UI сценарии safe и unsafe для parallel execution.
