# UI: качество и масштабирование, День 1 — Visual regression

## Зачем это нужно
- Не все UI-дефекты видны через DOM-assert или text checks.
- Visual regression помогает ловить сломанный layout и неожиданные сдвиги.
- Такие проверки особенно полезны на стабильных страницах и компонентах.

## Что важно понять
- Нужно понимать baseline, допустимые отличия и источники шума.
- Visual checks чувствительны к окружению, шрифтам и размеру окна.
- Применять их лучше точечно, а не на каждый экран проекта.

## Частые ошибки
- Снимки для нестабильных страниц с динамическим контентом.
- Отсутствие контроля baseline.
- Принятие любого диффа без анализа.

## Практика дня
Открой practice.py и перечисли, какие страницы подходят для visual regression лучше всего.
