# UI: качество и масштабирование, День 2 — Accessibility basics

## Зачем это нужно
- A11y checks помогают ловить реальные UX-проблемы, а не только compliance issues.
- Для QA это быстрый способ поднять качество UI без огромного аудита.
- Роли, labels и keyboard navigation часто ломаются незаметно.

## Что важно понять
- Базовый слой — role, label, accessible name и keyboard reachability.
- A11y-проверка полезна даже без полного аудита WCAG.
- Многие проблемы можно ловить уже на уровне локаторов и доступных имен.

## Частые ошибки
- Считать accessibility только дизайнерской темой.
- Проверять только aria-атрибуты без поведения.
- Игнорировать keyboard-only сценарии.
