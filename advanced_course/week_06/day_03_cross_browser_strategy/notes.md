# UI: качество и масштабирование, День 3 — Cross-browser strategy

## Зачем это нужно
- Одинаковый UI может вести себя по-разному в Chromium, Firefox и WebKit.
- Полный запуск всего набора везде дорог, поэтому нужна стратегия.
- Cross-browser coverage должна быть risk-based.

## Что важно понять
- Нужно выделять smoke matrix и расширенный matrix.
- Не каждый сценарий стоит гонять на всех браузерах.
- Проблемы layout, file upload и focus часто браузеро-зависимы.

## Частые ошибки
- Запускать всё везде без приоритетов.
- Проверять только один браузер и делать вывод про все.
- Отсутствие явного browser matrix в проекте.

## Практика дня
Открой practice.py и предложи browser matrix для smoke и regression.
