# Неделя 7, День 5 — Page Object Model

## Что такое POM
Page Object Model — это подход, при котором логика работы со страницей выносится в отдельный класс.

## Зачем это нужно
- тесты становятся короче и понятнее
- локаторы и действия хранятся в одном месте
- изменение страницы легче поддерживать

## Идея структуры
- в page object лежат локаторы и методы страницы
- в тесте лежит сценарий и проверки

## Пример мышления
Плохо:
- в тесте много локаторов и действий вперемешку

Лучше:
- `LoginPage.login(username, password)`
- `DashboardPage.open_profile()`

## Практические правила
- page object должен описывать страницу или её компонент
- не превращай page object в огромный хаотичный класс
- тест должен читатьcя как сценарий высокого уровня
