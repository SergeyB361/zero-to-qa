# Неделя 5, День 7 — Мини-проект: тесты функций

## Цель
Собрать небольшой набор тестов для функций из недель 1–2 и оформить их в виде понятного мини-проекта.

## Что здесь закрепляется
- `pytest`
- именование тестов
- `assert`
- AAA-подход
- базовые негативные сценарии

## Как подойти к проекту
1. сначала выбери 3-5 простых функций
2. потом определи, что у них является корректным результатом
3. затем добавь позитивные сценарии
4. после этого добавь граничные или негативные случаи

## Что здесь важно
- использовать `pytest`
- давать тестам понятные имена
- применять `assert`
- по возможности соблюдать AAA
- покрыть позитивные и негативные сценарии

## Как оценивать результат
Хороший мини-проект на этом этапе:
- запускается без ручных действий
- имеет понятные функции
- содержит читаемые тесты
- не перегружен лишней архитектурой

## Практика дня
Ориентируйся на `spec.md` и собери мини-набор тестов.
