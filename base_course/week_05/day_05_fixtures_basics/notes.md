# Неделя 5, День 5 — Фикстуры: основы

## Что такое fixture
Фикстура в `pytest` помогает подготовить данные или объекты для тестов.

## Зачем нужны фикстуры
- убрать дублирование
- централизовать подготовку данных
- сделать тесты чище

## Пример
```python
import pytest

@pytest.fixture
def user_data():
    return {"username": "anna", "role": "admin"}
```

## Как это используется
```python
def test_user_role(user_data):
    assert user_data["role"] == "admin"
```

PyTest сам подставит фикстуру по имени параметра.

## Где это полезно
- тестовые пользователи
- временные файлы
- объекты API-клиента
- конфигурация окружения

## Практические правила
- фикстура должна готовить данные, а не прятать всю логику теста
- имя фикстуры должно быть понятным
- не делай фикстуры слишком магическими на раннем этапе
