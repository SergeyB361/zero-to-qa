# Неделя 3, День 5 — Logging

## Что такое logging
`logging` — это стандартный модуль Python для записи сообщений о работе программы.

Он полезен, когда нужно:
- понимать, что делает программа,
- видеть ошибки и предупреждения,
- оставлять служебные сообщения для отладки,
- не заменять всё подряд `print()`.

## Зачем это нужно QA
В тестовых утилитах и автотестах logging помогает:
- видеть шаги сценария,
- фиксировать ошибки,
- быстрее разбирать падения,
- делать вывод программы понятнее.

## Уровни логов
- `logging.debug()` — подробности для отладки: значения переменных, промежуточные шаги
- `logging.info()` — нормальные события работы программы: запуск, успешное действие, чтение файла
- `logging.warning()` — что-то пошло неидеально, но программа ещё может работать дальше
- `logging.error()` — операция завершилась ошибкой
- `logging.critical()` — серьёзная ошибка, после которой программа или важный сценарий не может нормально продолжаться

## Базовый пример
```python
import logging

logging.basicConfig(level=logging.INFO)

logging.info("Program started")
logging.warning("Low disk space")
logging.error("Request failed")
```

## Формат сообщений
Можно настроить формат:
```python
logging.basicConfig(
    level=logging.INFO,
    format="%(levelname)s: %(message)s"
)
```

## Что важно
- `print()` — для обычного вывода пользователю
- `logging` — для служебной информации о работе программы
- для учебных скриптов достаточно `basicConfig(...)`
