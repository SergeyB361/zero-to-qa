# Неделя 3, День 7 — Мини-проект: CLI-анализатор тестовых артефактов

## Цель дня
Собрать небольшую CLI-утилиту, которая читает файл с тестовыми событиями и выводит полезную сводку.

## Что здесь закрепляется
- исключения
- type hints
- регулярные выражения
- logging
- pathlib
- разделение логики на функции

## Идея проекта
Есть текстовый файл `artifacts.log` с сообщениями вида:
- `INFO`
- `WARNING`
- `ERROR`
- `TEST PASSED`
- `TEST FAILED`

Программа должна:
- прочитать файл,
- посчитать общее число строк,
- посчитать количество `ERROR`,
- посчитать количество `PASSED` и `FAILED`,
- найти все числовые `id` через regex,
- вывести краткую сводку в консоль.

## Как лучше подойти
Работай по шагам:
1. сначала чтение файла
2. потом отдельные функции подсчёта
3. потом извлечение `id`
4. в конце — `main()` и печать итоговой сводки

Так проект проще проверять и отлаживать по частям.

## Что важно
- работать через `Path`
- разделить чтение файла, анализ и вывод
- не дублировать одинаковые проходы по данным без необходимости
- сделать понятную CLI-точку входа

## Критерии хорошего решения
- функции маленькие и делают одну вещь
- `main()` только собирает шаги вместе
- код не завязан на случайные глобальные переменные
- вывод соответствует `spec.md`
