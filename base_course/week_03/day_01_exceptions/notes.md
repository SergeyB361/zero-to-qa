# Неделя 3, День 1 — Исключения

## Зачем нужны исключения
Исключения позволяют корректно обрабатывать ошибки во время выполнения программы, а не просто падать с traceback.

Для QA и Python-утилит это важно, когда:
- пользователь вводит некорректные данные
- файла нет на диске
- произошла ошибка при вычислении
- данные нарушают бизнес-правила

## Базовая конструкция
```python
try:
    # код, где может быть ошибка
except ValueError:
    # что делать при этой ошибке
else:
    # выполнится, если ошибки не было
finally:
    # выполнится всегда
```

## Как читать эту конструкцию
- `try` — здесь может возникнуть ошибка
- `except` — обработка конкретной ошибки
- `else` — код для успешного сценария
- `finally` — код, который должен выполниться в любом случае

## Частые типы ошибок
- `ValueError` — неверное значение, например `int("abc")`
- `ZeroDivisionError` — деление на ноль
- `FileNotFoundError` — файл не найден
- `KeyError` — ключа нет в словаре

## `raise`
Иногда ошибку нужно создать вручную:

```python
def set_age(age):
    if age < 0:
        raise ValueError("age must be >= 0")
```

Это полезно, когда данные формально существуют, но не проходят бизнес-правило.

## Практические правила
- Лови конкретные ошибки (`except ValueError`, а не просто `except`).
- Не скрывай ошибку молча: выводи понятное сообщение.
- Не используй исключения как замену обычной логике `if` без причины.
- `finally` используй для действий, которые должны произойти всегда.
- Если данные некорректны по смыслу задачи, можно бросить ошибку вручную через `raise`.

## Мини-пример
```python
value = "10"

try:
    number = int(value)
    result = 100 / number
except ValueError:
    print("Нужно ввести целое число")
except ZeroDivisionError:
    print("Делить на ноль нельзя")
else:
    print("Результат:", result)
finally:
    print("Проверка завершена")
```
