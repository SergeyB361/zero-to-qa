## 1. Переменные

Переменная — это **имя, связанное с объектом в памяти**.
В Python не нужно объявлять тип заранее — интерпретатор определяет его автоматически при присваивании.

```python
x = 10          # x ссылается на объект int(10)
x = "hello"     # теперь x ссылается на объект str("hello")
```

Оператор `=` — это **присваивание**, не математическое равенство.
Он связывает имя с объектом, а не копирует значение в ячейку памяти.

**Соглашение об именах (PEP 8):**
- `snake_case` для переменных и функций: `user_name`, `test_result`
- Имя должно отражать смысл: `count`, не `c`
- Нельзя начинать с цифры, нельзя использовать дефис

## 2. Встроенные типы данных

### int
Целые числа произвольной точности (нет переполнения, как в C/Java).

```python
count = 42
negative = -7
big = 10 ** 100    # Python спокойно работает с такими числами
```

### float
Числа с плавающей точкой, стандарт IEEE 754 (64-bit double).

```python
rate = 98.6
pi = 3.14159

# Важно: деление / всегда возвращает float
print(10 / 2)    # 5.0, не 5
```

### str
Неизменяемая последовательность Unicode-символов.

```python
name = "Sergey"
path = 'C:\\Users\\test'
sql = """SELECT *
FROM users
WHERE active = 1"""   # многострочный литерал
```

Конкатенация и повторение:
```python
"Hello" + " " + "World"    # "Hello World"
"=" * 40                   # строка из 40 символов "="
```

### bool
Подкласс `int`. `True == 1`, `False == 0` — это не метафора, это буквально так.

```python
print(True + True)     # 2
print(True * 5)        # 5
print(False == 0)      # True
```

### None
Singleton-объект типа `NoneType`. Используется как явное "отсутствие значения".
Не то же самое, что `0`, `""` или `False`.

```python
result = None
print(result is None)    # True  ← правильный способ проверки
print(result == None)    # True  ← работает, но не рекомендуется
```

## 3. Проверка типа: type() и isinstance()

```python
x = 42
print(type(x))              # <class 'int'>
print(type(x) == int)       # True

# isinstance() — предпочтительный способ, учитывает наследование
print(isinstance(x, int))       # True
print(isinstance(x, (int, float)))  # True — проверка на несколько типов
```

## 4. Арифметические операторы

```python
7 + 3     # 10   — сложение
7 - 3     # 4    — вычитание
7 * 3     # 21   — умножение
7 / 2     # 3.5  — деление, всегда float
7 // 2    # 3    — целочисленное деление (отбрасывает дробь)
7 % 2     # 1    — остаток от деления (modulo)
2 ** 8    # 256  — возведение в степень
```

`//` и `%` связаны математически: `a == (a // b) * b + (a % b)` — всегда верно.

## 5. Приведение типов (Type Casting)

Python — язык с **динамической, но строгой** типизацией.
Неявное приведение не выполняется — нужно делать явно.

```python
# Явное приведение
int("42")        # 42
float("3.14")    # 3.14
str(100)         # "100"
bool(0)          # False

# Что считается False (falsy values)
bool(0)          # False
bool(0.0)        # False
bool("")         # False
bool(None)       # False
bool([])         # False — пустой список

# Всё остальное — True
bool(1)          # True
bool(-1)         # True
bool("0")        # True — непустая строка!
```

Попытка склеить `str` и `int` без приведения — `TypeError`:
```python
"Версия: " + 3        # ❌ TypeError
"Версия: " + str(3)   # ✅ "Версия: 3"
```

## 6. f-строки (f-strings, PEP 498)

Введены в Python 3.6. Самый современный и читаемый способ форматирования.

```python
name = "Sergey"
passed = 47
total = 50

print(f"Тестировщик: {name}")
print(f"Результат: {passed}/{total} = {passed/total:.1%}")
# Тестировщик: Sergey
# Результат: 47/50 = 94.0%
```

Внутри `{}` можно использовать любое выражение:
```python
print(f"{'PASS' if passed == total else 'FAIL'}")
print(f"{2 ** 10 = }")    # Python 3.8+: выводит выражение и результат
```

Форматирование чисел:
```python
f"{3.14159:.2f}"    # "3.14"    — 2 знака после точки
f"{1000000:,}"      # "1,000,000" — разделитель тысяч
f"{0.947:.1%}"      # "94.7%"   — процент
f"{255:08b}"        # "11111111" — двоичный, 8 знаков
```

## 7. Идентичность объектов: is и id()

Каждый объект в памяти имеет уникальный идентификатор — `id()`.

```python
x = 42
y = x

print(id(x))        # например: 140712345678
print(id(y))        # тот же адрес — y ссылается на тот же объект
print(x is y)       # True

y = 43
print(x is y)       # False — теперь разные объекты
```

**`==` vs `is`:**

```python
a = [1, 2, 3]
b = [1, 2, 3]

print(a == b)    # True  — одинаковые значения
print(a is b)    # False — разные объекты в памяти

c = a
print(a is c)    # True  — одна и та же ссылка
```

- `==` сравнивает **значения**
- `is` сравнивает **идентичность** (один ли объект)

На практике `is` используют только с `None`, `True`, `False`:
```python
result is None     # ✅ правильно (PEP 8)
result == None     # ⚠️  работает, но не рекомендуется
```

## 8. Инструменты исследования: dir() и help()

Незаменимы при работе в незнакомом коде или когда забыл название метода.

```python
# dir() — список всех атрибутов и методов объекта
dir("hello")     # покажет: capitalize, count, find, replace, split...
dir(42)          # покажет методы int: bit_length, conjugate...

# Фильтруем служебные методы (начинающиеся на __)
[m for m in dir("") if not m.startswith("_")]

# help() — документация по объекту или методу
help(str)           # полная документация по строкам
help("".split)      # только по методу split
help(int)           # по целым числам
```

Эти функции доступны без импорта — встроены в Python.

## 9. Ключевые слова vs встроенные функции

В Python есть два принципиально разных понятия.

**Ключевые слова (keywords)** — часть грамматики языка. Не являются объектами,
не имеют типа, нельзя передать в функцию или присвоить переменной.

```python
import keyword
print(keyword.kwlist)
# ['False', 'None', 'True', 'and', 'as', 'assert', 'async', 'await',
#  'break', 'class', 'continue', 'def', 'del', 'elif', 'else',
#  'except', 'finally', 'for', 'from', 'global', 'if', 'import',
#  'in', 'is', 'lambda', 'nonlocal', 'not', 'or', 'pass', 'raise',
#  'return', 'try', 'while', 'with', 'yield']

keyword.iskeyword("is")     # True
keyword.iskeyword("dir")    # False

type(if)    # SyntaxError — интерпретатор даже не дойдёт до выполнения
x = for     # SyntaxError
```

**Встроенные функции (builtins)** — обычные объекты типа
`builtin_function_or_method`, просто доступны без импорта.

```python
type(print)    # <class 'builtin_function_or_method'>
type(len)      # <class 'builtin_function_or_method'>
type(dir)      # <class 'builtin_function_or_method'>

# Это объекты — их можно передавать как значения
f = print
f("hello")     # работает

# Полный список встроенных имён
import builtins
print(dir(builtins))
```

В Python действует принцип: **всё что имеет тип — объект**.
Числа, строки, функции, классы, модули — всё объекты.
Ключевые слова — единственное исключение.

> ⚠️ Не называй переменные именами встроенных функций:
> `list`, `dict`, `type`, `id`, `input`, `print`, `dir`, `len` — всё это
> встроенные имена. Переопределить можно, но это сломает их работу.

```python
list = [1, 2, 3]      # ← теперь list — не функция, а переменная
list([4, 5, 6])       # TypeError: 'list' object is not callable
```

## 10. Изменяемые и неизменяемые типы (preview)

> Тема полностью раскрывается в Неделю 2 на примерах списков, кортежей и словарей.
> Сейчас — ориентир для понимания текущих типов.

**Неизменяемые (immutable)** — после создания объект изменить нельзя.
При любой "модификации" создаётся **новый объект**, старый остаётся нетронутым.

```python
# str — неизменяемый
x = "hello"
print(id(x))        # например: 2001234

x += " world"       # не модификация, а создание нового объекта
print(id(x))        # другой адрес — это уже другой объект
```

Неизменяемые типы из уже изученных: `int`, `float`, `str`, `bool`, `None`

**Изменяемые (mutable)** — объект меняется на месте, `id` не меняется.
Изменяемые типы — `list`, `dict`, `set` — будем изучать в Неделю 2.

```python
# list — изменяемый (забегаем вперёд для понимания разницы)
a = [1, 2, 3]
print(id(a))        # например: 3001234

a.append(4)         # модификация на месте
print(id(a))        # тот же адрес — это тот же объект
```

Почему это важно — станет ясно в Неделю 2 когда увидим неожиданное поведение:

```python
# Опасный сценарий с изменяемым типом (preview)
a = [1, 2, 3]
b = a               # b ссылается на тот же объект!
b.append(4)
print(a)            # [1, 2, 3, 4] ← a тоже изменился!
```
