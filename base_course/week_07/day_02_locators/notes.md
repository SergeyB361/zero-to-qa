# Неделя 7, День 2 — Локаторы

## Что такое локатор
Локатор — это способ найти элемент на странице.

## Частые варианты
- по тексту
- по роли
- по label
- по CSS
- по XPath

## Почему локаторы важны
Если локатор нестабилен, тест будет падать не из-за бага в продукте, а из-за слабой автоматизации.

## Хорошая стратегия
Предпочитай локаторы, которые ближе к пользовательскому смыслу:
- `get_by_role`
- `get_by_label`
- `get_by_text`

CSS и XPath тоже нужны, но их стоит использовать осознанно.

## Практические правила
- не завязывайся на хрупкие классы и случайную структуру DOM без необходимости
- локатор должен быть читаемым
- по коду теста должно быть понятно, какой элемент ищется
