# Неделя 4, День 3 — Наследование

## Что даёт наследование
Наследование позволяет создавать новый класс на основе существующего и переиспользовать код.

## Родительский и дочерний классы
- родительский класс — базовый класс
- дочерний класс — расширяет или изменяет поведение родителя

Если `Admin` наследуется от `User`, то объект `Admin` получает атрибуты и методы `User`, если они не были переопределены.

## Базовый пример
```python
class User:
    def __init__(self, name):
        self.name = name

class Admin(User):
    def ban(self):
        print(f"{self.name} banned user")
```

Здесь `Admin` — это тоже `User`, но с дополнительным поведением.

## Когда наследование уместно
Наследование подходит, когда есть отношение:
- `Admin` является `User`
- `Manager` является `Employee`

Если такого отношения нет, лучше подумать о композиции или обычных функциях.

## `super()`
Через `super()` можно вызвать логику родительского класса:

```python
class Admin(User):
    def __init__(self, name, level):
        super().__init__(name)
        self.level = level
```

Это позволяет не дублировать код инициализации родителя.

## Переопределение
Дочерний класс может заменить метод родителя своей реализацией.

Пример:
```python
class User:
    def describe(self):
        return "regular user"

class Admin(User):
    def describe(self):
        return "admin user"
```

Это называется переопределением метода.

## Практические правила
- Используй наследование, когда есть отношение "является".
- Если нужна просто общая утилита, иногда лучше композиция, а не наследование.
- Не дублируй код `__init__`, если можно вызвать `super()`.
- Не строй слишком глубокие цепочки наследования без реальной необходимости.
