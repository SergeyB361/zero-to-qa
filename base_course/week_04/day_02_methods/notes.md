# Неделя 4, День 2 — Методы

## Зачем нужны методы
Класс хранит данные, а методы описывают, что объект или класс умеет делать с этими данными.

Просто правило:
- атрибуты отвечают за состояние
- методы отвечают за поведение

Пример:
- `self.name` — данные объекта
- `greet()` — действие объекта

## Что такое метод
Метод — это функция, объявленная внутри класса.

Главный вопрос при выборе метода такой:
- нужен доступ к конкретному объекту?
- нужен доступ ко всему классу?
- или это просто вспомогательная функция рядом с классом?

От ответа зависит, какой вид метода нужен.

## Три основных вида методов

### 1. Обычный метод
Обычный метод работает с конкретным объектом.

Сигнатура:
```python
class User:
    def greet(self):
        return f"Hello, {self.name}"
```

Важно:
- декоратор не нужен
- первый параметр — `self`
- `self` указывает на конкретный объект
- такой метод может читать и менять данные объекта

Когда использовать:
- если нужен `self.name`, `self.value`, `self.status`
- если метод работает с состоянием одного объекта

Пример смысла:
- пользователь приветствует
- счётчик увеличивает своё значение
- сообщение возвращает короткую версию текста

### 2. `@classmethod`
`@classmethod` работает с самим классом, а не с отдельным объектом.

Сигнатура:
```python
class User:
    total = 0

    @classmethod
    def get_total(cls):
        return cls.total
```

Важно:
- нужен декоратор `@classmethod`
- первый параметр — `cls`
- `cls` указывает на сам класс
- метод работает с общими данными класса

Когда использовать:
- если метод читает или меняет атрибуты класса
- если нужен альтернативный способ создать объект
- если нужен счётчик или общая логика для всех объектов

Пример смысла:
- вернуть число созданных объектов
- создать объект через фабричный метод

### 3. `@staticmethod`
`@staticmethod` логически относится к классу, но не использует ни объект, ни класс.

Сигнатура:
```python
class User:
    @staticmethod
    def is_valid_name(name):
        return len(name) >= 2
```

Важно:
- нужен декоратор `@staticmethod`
- `self` не нужен
- `cls` не нужен
- метод не зависит от состояния объекта и класса

Когда использовать:
- если это небольшая проверка или утилита
- если функция тематически относится к классу
- если не нужен доступ ни к объекту, ни к классу

Пример смысла:
- проверить имя
- проверить возраст
- проверить формат email

## Сравнение методов

### Что стоит в параметрах
- обычный метод: `self`
- `@classmethod`: `cls`
- `@staticmethod`: ничего специального

### С чем работает метод
- обычный метод: с конкретным объектом
- `@classmethod`: с классом целиком
- `@staticmethod`: только с переданными аргументами

### Нужен ли декоратор
- обычный метод: нет
- `@classmethod`: да
- `@staticmethod`: да

## Как понять, какой метод нужен
Используй такое правило выбора:

1. Если метод использует данные объекта — нужен обычный метод.
2. Если метод использует данные класса — нужен `@classmethod`.
3. Если метод не использует ни объект, ни класс — нужен `@staticmethod`.
4. Если функция вообще не связана с классом по смыслу — лучше вынести её вне класса.

## Что Python передаёт автоматически
При вызове ты не передаёшь `self` и `cls` вручную.

Пример:
```python
user = User("Ann")
user.greet()
```

Здесь Python сам подставляет объект в `self`.

И так же для класса:
```python
User.get_total()
```

Здесь Python сам подставляет класс в `cls`.

А у `@staticmethod` ничего автоматически не подставляется, кроме обычных аргументов, которые ты передал сам.

## Один класс с тремя видами методов
```python
class User:
    total = 0

    def __init__(self, name):
        self.name = name
        User.total += 1

    def greet(self):
        return f"Hello, {self.name}"

    @classmethod
    def get_total(cls):
        return cls.total

    @staticmethod
    def is_valid_name(name):
        return len(name) >= 2
```

Как это читается:
- `greet()` работает с конкретным пользователем
- `get_total()` работает с классом `User`
- `is_valid_name()` просто выполняет проверку

## Частые ошибки
- писать `@staticmethod`, хотя внутри нужен `self`
- писать `@classmethod`, хотя метод работает только с объектом
- складывать в класс функции, которые вообще не связаны с его ролью

## Практические правила
- Если метод меняет состояние объекта, это почти всегда обычный метод.
- Если метод опирается на общий счётчик или фабрику объекта, это `@classmethod`.
- Если метод только проверяет входные данные, это часто `@staticmethod`.
- Не используй `@staticmethod` просто потому, что можно. Иногда обычная функция вне класса лучше.
