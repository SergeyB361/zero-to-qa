# Неделя 4, День 7 - Мини-проект: User / Admin

## Цель дня
Собрать небольшую систему пользователей и закрепить темы недели:
- классы и объекты
- методы
- наследование
- инкапсуляция

## Что ты собираешь
Мини-систему из нескольких классов, где:
- есть базовый класс `User`
- есть дочерний класс `Admin`
- у объектов есть данные и методы
- часть поведения наследуется, а часть добавляется в дочернем классе

## Мини-архитектура
- базовый класс `User`
- дочерний класс `Admin`
- методы для описания пользователя и проверки доступа

## Как подойти к проекту
Работай по шагам:
1. сначала реализуй базовый класс `User`
2. потом добавь нужные методы
3. после этого создай `Admin` через наследование
4. только затем добавляй проверки и дополнительную логику

Так проще не запутаться и проверять проект по частям.

## Что важно
- аккуратный `__init__`
- использование `super()` там, где это уместно
- понятное разделение ответственности между классами
- простые и читаемые имена атрибутов и методов

## Важный организационный момент
`users_system.py` специально сделан как рабочий scaffold:
- файл запускается до завершения проекта;
- внутри есть TODO, которые ты постепенно заменяешь своей логикой;
- итоговая цель описана в `spec.md`.

Это сделано специально, чтобы мини-проект не падал на старте и его можно было развивать по шагам.

## Как оценивать своё решение
Проверь себя:
- у каждого класса есть понятная роль
- нет лишнего дублирования кода
- наследование используется по смыслу
- интерфейс классов понятен при чтении

## Следующий шаг
Открой `spec.md` и реализуй требования в `users_system.py`.
