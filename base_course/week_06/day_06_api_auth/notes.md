# Неделя 6, День 6 — Авторизация в API

## Что такое авторизация
Авторизация в API определяет, можно ли клиенту выполнять конкретный запрос после подтверждения прав доступа.

## Частые варианты
- токен в `Authorization` header
- API key
- session/cookie
- Basic Auth

## Частый пример
```python
headers = {"Authorization": "Bearer <token>"}
response = requests.get(url, headers=headers)
```

## Что важно для тестирования
- запрос без токена
- запрос с валидным токеном
- запрос с невалидным токеном
- проверка статусов `401` и `403`

## Практические правила
- не храни реальные секреты прямо в коде
- авторизацию полезно выносить в конфиг или helper/client
- проверяй не только успешный, но и запрещённый сценарий
