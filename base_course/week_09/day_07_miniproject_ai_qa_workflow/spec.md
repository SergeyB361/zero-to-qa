# ТЗ: AI-assisted QA workflow

## Что сделать
Собери мини-проект, который показывает безопасное и полезное применение AI на реальном файле этого репозитория.

## Что должно быть внутри
1. `playbook.md` с правилами использования AI.
2. `prompts.md` с 5-10 полезными промптами.
3. `review_checklist.md` с верификацией AI-результатов.
4. Один практический результат работы AI:
- `test_*.py`
- или `review_notes.md`
- или `debug_notes.md`

## Что покрыть по смыслу
- генерация test cases
- генерация unit/API/UI тестов
- code review и debug
- agent workflows
- ограничения и риски

## Рекомендуемые реальные цели
- `week_03/day_07_miniproject_cli_artifacts/artifact_cli.py`
- `week_04/day_07_miniproject_user_admin/users_system.py`
- одна простая функция из `week_01` или `week_02`

## Обязательное требование
Нужно не только попросить AI о помощи, но и отдельно зафиксировать:
- что AI сделал правильно
- что AI сделал неправильно
- что пришлось исправить вручную

## Что использовать как источники при проверке
- материалы из [references.md](C:/Users/serge/zero-to-qa/week_09/references.md)
- `pytest` docs для unit-тестов
- `Playwright` docs для UI-тестов
- официальные материалы по prompt engineering и evals для оценки качества AI-ответа

## Цель
Получить не просто теорию, а рабочий шаблон того, как использовать AI в ежедневной инженерной практике.
