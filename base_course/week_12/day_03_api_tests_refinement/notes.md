# Неделя 12, День 3 — API-тесты: доработка

## Зачем нужна доработка API-слоя
Финальный API-блок должен быть не только рабочим, но и аккуратным: понятные клиенты, стабильные проверки, читаемые сценарии.

## Что обычно улучшают
- структуру тестов
- повторное использование кода
- понятность проверок
- обработку конфигов и данных

## Практические правила
- удали лишние повторы
- укрепи ключевые сценарии
- сделай так, чтобы по тесту было видно бизнес-смысл проверки

## Практика дня
Открой `practice.py` и оцени, какие части API-набора требуют чистки или усиления.
