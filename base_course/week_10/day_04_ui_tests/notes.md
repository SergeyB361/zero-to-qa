# Неделя 10, День 4 — UI-тесты

## Роль UI-тестов в проекте
UI-тесты проверяют пользовательский слой. Они обычно дороже и медленнее API-тестов, но незаменимы для ключевых end-to-end сценариев.

## Что важно в хорошем UI-блоке
- понятные page objects
- устойчивые локаторы
- читаемые сценарии
- разумное покрытие, а не попытка проверить весь интерфейс подряд

## Практические правила
- UI-тесты должны покрывать действительно важные пользовательские флоу
- если сценарий можно надёжнее проверить на API или unit-уровне, это стоит учитывать
- UI-тесты особенно выигрывают от хорошей структуры проекта
