# Неделя 10, День 3 — API-тесты

## Зачем выделять API-тесты в проекте
API-тесты позволяют быстро и стабильно проверять серверную логику, не завися от интерфейса браузера.

## Что важно в хорошем API-блоке
- понятная структура
- переиспользуемые clients/helpers
- ясные проверки статуса, JSON и бизнес-смысла

## Как это связано с архитектурой проекта
API-тесты не должны жить как набор случайных файлов. Им нужна понятная организация: endpoints, clients, config, test data.

## Практические правила
- API-тесты должны быть быстрее и стабильнее UI-тестов
- проверяй не только статус-код, но и смысл ответа
- переиспользуемую логику выноси из тестов
