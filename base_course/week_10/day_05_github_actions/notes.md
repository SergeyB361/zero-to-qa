# Неделя 10, День 5 — GitHub Actions

## Что это такое
GitHub Actions — это механизм CI/CD внутри GitHub, который позволяет запускать процессы автоматически, например тесты при `push`.

## Зачем это нужно QA
- тесты проверяются не только локально
- команда видит результат в одном месте
- проект выглядит профессиональнее

## Что обычно делает workflow
- устанавливает зависимости
- запускает тесты
- сохраняет результат или артефакты

## Практические правила
- workflow должен быть понятным и воспроизводимым
- не стоит сразу делать слишком сложный pipeline
- сначала важна надёжность базового запуска
