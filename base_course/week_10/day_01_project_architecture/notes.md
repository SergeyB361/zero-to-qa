# Неделя 10, День 1 — Архитектура проекта

## Зачем нужна архитектура
Когда проект растёт, без структуры он быстро становится хаотичным. Архитектура нужна, чтобы код тестов, клиентов, page objects и утилит был понятен и поддерживаем.

## Что обычно распределяют по папкам
- тесты
- page objects
- API clients
- utils
- config
- test data

## Хорошая структура проекта
Хорошая структура делает понятным:
- где лежит логика
- где лежат тесты
- где лежат данные и настройки

## Практические правила
- не складывай всё в один файл
- имена папок и модулей должны отражать роль
- структура должна помогать поддержке проекта, а не быть декоративной
