# Неделя 11, День 6 — Структура проекта

## Почему структура снова важна
К этому моменту у проекта уже может быть много тестов, фикстур, page objects, API clients и утилит. Без структуры поддержка начнёт замедляться.

## Что обычно организуют
- папки тестов
- `conftest.py`
- конфигурационные файлы
- page objects / clients / utils
- test data

## Признаки хорошей структуры
- новый человек быстро понимает, где что лежит
- тесты не зависят от случайного порядка файлов
- переиспользуемый код вынесен из тестов

## Практические правила
- структура должна помогать навигации
- не стоит создавать много уровней папок без причины
- названия модулей должны быть стабильными и понятными
