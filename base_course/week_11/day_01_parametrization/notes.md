# Неделя 11, День 1 — Параметризация

## Что такое параметризация
Параметризация позволяет запускать один и тот же тест на разных наборах входных данных.

## Зачем это нужно
- убрать дублирование
- проверять несколько сценариев одним шаблоном теста
- держать тесты короче и понятнее

## Базовая идея
```python
import pytest

@pytest.mark.parametrize("a, b, expected", [
    (1, 2, 3),
    (2, 3, 5),
])
def test_sum(a, b, expected):
    assert a + b == expected
```

## Когда это особенно полезно
- одна функция, много наборов данных
- позитивные и негативные сценарии одного поведения
- проверки на границы и разные комбинации

## Практические правила
- параметризация должна упрощать тест, а не делать его нечитаемым
- имена параметров должны быть понятными
- если сценарии слишком разные по смыслу, иногда лучше оставить отдельные тесты
