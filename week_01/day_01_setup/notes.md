## Шаг 2 — Виртуальное окружение (venv)

**Зачем:** каждый проект хранит свои библиотеки отдельно, не пересекаясь с другими проектами.

```bash
# Создать окружение
python -m venv venv

# Активировать (Windows)
venv\Scripts\activate

# Проверить — в терминале должно появиться (venv) слева
python --version

# Деактивировать когда не нужно
deactivate
```

После активации терминал выглядит так:
```
(venv) PS C:\Users\serge\gh2>
```

## Шаг 4 — Git: первые команды

**Git** — система контроля версий. Сохраняет историю изменений кода.

```bash
# Инициализировать репозиторий
git init

# Посмотреть состояние файлов
git status

# Добавить все файлы в индекс
git add .

# Сохранить снимок (коммит)
git commit -m "init: старт курса Путь с нуля до QA"

# Посмотреть историю коммитов
git log --oneline
```

## Шаг 5 — .gitignore

Файл `.gitignore` говорит Git что **не отслеживать**.
Уже создан в корне проекта:

```
venv/           ← тысячи файлов окружения — не нужны в git
__pycache__/    ← автогенерируемые файлы Python
*.pyc           ← скомпилированные файлы
.env            ← секреты и пароли — никогда в git!
```
