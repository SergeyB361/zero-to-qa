# Неделя 12, День 6 — README и документация

## Зачем это важно
Документация — это часть качества проекта. Если проект нельзя быстро понять и запустить, его ценность падает.

## Что должен объяснять README
- что это за проект
- какой стек используется
- как установить зависимости
- как запускать тесты
- что именно покрывает проект

## Что ещё полезно
- структура проекта
- примеры команд
- описание отчётов
- ограничения или допущения

## Практические правила
- README должен быть полезным, а не декоративным
- не прячь критичную информацию только в коде
- документация должна быть актуальной
