# Неделя 1, День 4 - Линейный поиск

## Что это такое
Линейный поиск - это самый простой способ найти элемент.

Идея:
- идти по массиву слева направо;
- сравнивать каждый элемент с искомым значением;
- остановиться, когда найдено совпадение.

## Когда он работает
Линейный поиск работает в любом списке:
- отсортированном;
- неотсортированном;
- со строками;
- с числами;
- с объектами.

## Сложность
В худшем случае нужно пройти весь список.
Поэтому сложность обычно `O(n)`.

## Когда это нормально
Линейный поиск нормален, если:
- данных мало;
- массив не отсортирован;
- поиск выполняется редко;
- важна простота решения.

## Когда этого мало
Если поиск выполняется часто и данных много, обычно ищут более эффективный подход:
- бинарный поиск в отсортированных данных;
- `set` или `dict` для быстрого membership lookup.

## Типовые варианты задачи
- вернуть `True/False`, найден ли элемент;
- вернуть индекс первого вхождения;
- вернуть все индексы;
- вернуть количество вхождений.

## Типичные ошибки
1. Не обрабатывать случай, когда элемент не найден.
2. Возвращать неправильный индекс.
3. Забывать выйти из цикла, если нужен только первый результат.
4. Использовать линейный поиск там, где уже лучше `set`.
