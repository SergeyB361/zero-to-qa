# Неделя 1, День 1 - Что такое алгоритм

## Что такое алгоритм
Алгоритм - это точная последовательность шагов, которая решает задачу.

Примеры задач:
- найти максимум в списке;
- проверить, есть ли число в массиве;
- посчитать количество чётных элементов;
- отсортировать данные.

## У хорошего алгоритма есть 4 свойства
1. **Понятные шаги** - должно быть ясно, что делать на каждом этапе.
2. **Конечность** - алгоритм обязан завершаться.
3. **Результат** - после выполнения должен быть ответ.
4. **Повторяемость** - при одинаковом входе логика работает одинаково.

## Вход и выход
Обычно алгоритм можно описать так:
- входные данные;
- последовательность действий;
- выходные данные.

Пример:
- вход: список чисел;
- действия: пройти по списку и найти наибольшее;
- выход: максимальное число.

## Алгоритм и программа - не одно и то же
- алгоритм - это идея решения;
- программа - это реализация этой идеи на языке программирования.

Сначала нужно понимать решение как алгоритм, а уже потом писать код.

## Пример алгоритма без кода
Задача: найти сумму всех положительных чисел.

Шаги:
1. взять сумму = 0;
2. пройти по всем элементам списка;
3. если элемент больше 0, прибавить его к сумме;
4. после прохода вернуть сумму.

## Почему это важно для разработчика и QA
Если ты мыслишь алгоритмами, ты лучше:
- понимаешь, почему код работает;
- находишь лишние шаги;
- замечаешь ошибки в логике;
- умеешь сравнивать два решения одной задачи.

## Типичные ошибки новичка
1. Сразу писать код без плана.
2. Смешивать входные данные, шаги и результат.
3. Не думать о случаях вроде пустого списка.
4. Писать решение, которое работает только на одном примере.

## Как тренироваться
Перед кодом отвечай на 3 вопроса:
1. Что приходит на вход?
2. Что должно получиться на выходе?
3. Какие шаги между ними?
