# Неделя 5, День 4 - Sliding Window: база

## Что это за паттерн
Sliding window - это способ работать с **непрерывным отрезком** массива или строки.

Окно движется по данным слева направо.

## Когда этот паттерн полезен
- сумма на каждом окне длины `k`;
- максимум/минимум на окне;
- количество уникальных на окне;
- поиск подстроки или диапазона с условием.

## Базовая идея fixed-size window
Если окно имеет фиксированный размер `k`, то не нужно пересчитывать всё заново.

Можно:
- добавить новый элемент справа;
- убрать старый элемент слева.

## Почему это лучше
Наивный подход часто делает много лишней работы.
Sliding window позволяет переиспользовать уже посчитанное.

## Пример
Нужно найти максимальную сумму подмассива длины `3`.

Вместо отдельного подсчёта для каждого окна можно:
- посчитать сумму первого окна;
- потом обновлять её за `O(1)` при сдвиге.

## Типичные ошибки
1. Путать окно и произвольный набор элементов.
2. Неправильно обновлять левую границу.
3. Пересчитывать окно заново на каждом шаге.
4. Плохо обработать случаи `k > len(items)`.
