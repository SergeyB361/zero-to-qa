# Неделя 3, День 3 - Сортировка слиянием

## Почему нужна ещё одна сортировка
Простые сортировки `O(n^2)` слишком медленные на больших входах.

Нужен более сильный подход.
Один из классических вариантов - merge sort.

## Идея
Merge sort использует подход divide and conquer:
1. делим массив пополам;
2. рекурсивно сортируем левую часть;
3. рекурсивно сортируем правую часть;
4. сливаем две отсортированные части в одну.

## Ключевая операция - merge
Если у тебя уже есть два отсортированных массива, их можно слить в один отсортированный проходом двумя указателями.

Это базовая и очень важная идея.

## Сложность
Merge sort работает за `O(n log n)`.

Это значительно лучше, чем `O(n^2)`.

## Минус
Обычно merge sort требует дополнительную память для слияния.

То есть алгоритм быстрый по времени, но не всегда самый экономный по памяти.

## Где это полезно
- когда нужна стабильная сортировка;
- когда важно гарантированное `O(n log n)`;
- как классический пример divide and conquer.

## Типичные ошибки
1. Плохо написать функцию слияния.
2. Потерять элементы при объединении хвостов.
3. Смешать индексы и значения.
4. Забыть базовый случай рекурсии.
