# Неделя 3, День 5 - Когда использовать built-in sorted()

## Главная мысль
В реальном Python чаще всего не нужно писать сортировку руками.

Обычно правильный выбор - `sorted()` или `.sort()`.

## Почему
Встроенная сортировка Python:
- быстрая;
- стабильная;
- хорошо протестированная;
- поддерживает `key` и `reverse`.

Алгоритм под капотом называется Timsort.

## Чем хорош `sorted()`
- возвращает новый список;
- не меняет исходную коллекцию;
- работает с любыми итерируемыми объектами.

## Чем хорош `.sort()`
- сортирует список на месте;
- не создаёт новый список;
- работает только для `list`.

## `key`
Очень важный инструмент.

Позволяет сортировать не только по самим элементам, но и по вычисляемому признаку.

Примеры:
- длина строки;
- второй элемент кортежа;
- поле словаря;
- атрибут объекта.

## Стабильность сортировки
Стабильная сортировка сохраняет относительный порядок равных элементов.

Это важно, когда:
- сортируешь в несколько этапов;
- есть вторичные и третичные критерии.

## Когда писать сортировку руками всё же полезно
- для обучения;
- на алгоритмических интервью;
- если задача прямо просит реализовать конкретный алгоритм.

## Практическое правило
Если задача прикладная и язык - Python, начинай с вопроса:
- можно ли решить это через `sorted(..., key=...)`?

Очень часто ответ: да.
