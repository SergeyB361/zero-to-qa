# ТЗ: Search and Sort Lab

Реализуй проект в `search_sort_lab.py`.

## Что должно быть в MVP
1. `linear_search(items, target)`
2. `binary_search(items, target)` для отсортированного списка
3. `merge_sort(items)` или `quick_sort(items)`
4. `main()`, который показывает полный demo flow

## Demo flow
1. Создай исходный неотсортированный список.
2. Покажи результат линейного поиска target.
3. Отсортируй список.
4. Покажи отсортированную версию.
5. Выполни бинарный поиск того же target.
6. Выведи результат в понятном виде.

## Пример допустимого вывода
```text
Original: [7, 2, 9, 1, 5]
Linear search for 5: 4
Sorted: [1, 2, 5, 7, 9]
Binary search for 5: 2
```

## Опционально
- показать поведение, когда элемент не найден;
- сравнить несколько target-значений;
- добавить built-in `sorted()` как дополнительную проверку.

## Критерии готовности
- файл запускается без ошибок;
- логика поиска и сортировки разделена по функциям;
- бинарный поиск применяется только к отсортированным данным;
- вывод читается без открытия кода.
