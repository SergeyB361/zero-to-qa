# Неделя 3, День 1 - Бинарный поиск

## Что это такое
Бинарный поиск - это алгоритм поиска в **отсортированном** массиве.

Идея:
- не идти по элементам подряд;
- смотреть на середину диапазона;
- отбрасывать половину данных за один шаг.

## Главное условие
Бинарный поиск работает только тогда, когда данные уже отсортированы.

Это ключевой момент.
Если массив не отсортирован, бинарный поиск применять нельзя.

## Как он работает
1. Берём левую и правую границу.
2. Находим середину.
3. Сравниваем середину с target.
4. Если середина меньше target, идём вправо.
5. Если середина больше target, идём влево.
6. Повторяем, пока диапазон не исчезнет.

## Почему он быстрый
На каждом шаге мы отбрасываем половину массива.

Поэтому сложность получается `O(log n)`.

Это намного лучше, чем линейный поиск `O(n)`, если:
- массив большой;
- поиск выполняется часто;
- данные уже отсортированы.

## Пример
Ищем `19` в массиве:

```python
[3, 7, 10, 14, 19, 25, 31]
```

- середина -> `14`
- `19 > 14`, идём вправо
- следующая середина -> `25`
- `19 < 25`, идём влево
- остаётся `19`

## Что может возвращать функция
Частые варианты:
- индекс найденного элемента;
- `-1`, если не найдено;
- `True/False`;
- позицию вставки.

## Где это реально используют
- поиск в отсортированном массиве;
- lower/upper bound;
- задачи на позицию вставки;
- поиск по ответу в некоторых задачах.

## Типичные ошибки
1. Применять бинарный поиск к неотсортированным данным.
2. Неправильно обновлять границы.
3. Застревать в бесконечном цикле.
4. Путать индекс и значение.

## Практическое правило
Если видишь:
- отсортированный массив;
- поиск элемента, границы или позиции,

то бинарный поиск - один из первых кандидатов.
