# Неделя 3, День 2 - Простые сортировки

## Зачем их изучать
В реальном Python почти всегда используют `sorted()` или `.sort()`.

Но простые сортировки всё равно важны, потому что они учат:
- как работают перестановки элементов;
- почему некоторые алгоритмы медленные;
- как сравнивать сортировки по сложности.

## Какие сортировки рассматриваем
- bubble sort
- selection sort
- insertion sort

## Bubble sort
Идея:
- проходим по массиву;
- сравниваем соседние элементы;
- меняем местами, если они в неверном порядке.

Большие элементы постепенно “всплывают” вправо.

## Selection sort
Идея:
- на каждом шаге ищем минимальный элемент в оставшейся части;
- ставим его на текущую позицию.

## Insertion sort
Идея:
- строим отсортированную часть массива слева;
- вставляем очередной элемент в правильное место.

## Сложность
Эти сортировки обычно имеют `O(n^2)`.

Это делает их плохим выбором для больших данных.

## Почему они всё же полезны
- просты в понимании;
- помогают увидеть цену вложенных циклов;
- полезны как учебная база перед более сильными сортировками.

## Где их можно использовать
На практике редко.
Обычно только:
- для обучения;
- для очень маленьких данных;
- в задачах, где важна простота реализации.

## Типичные ошибки
1. Сравнивать не те элементы.
2. Ломать индексы во внутренних циклах.
3. Забывать, что сортировка может мутировать исходный список.
4. Применять `O(n^2)`-решение к большим входам без причины.
