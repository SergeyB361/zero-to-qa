# Неделя 4, День 2 - Подсчёт частот

## Что это за паттерн
Очень много задач сводятся к тому, чтобы посчитать, сколько раз встречается каждое значение.

Это и есть frequency counting.

## Наивный подход
Можно для каждого элемента снова проходить по всему массиву и считать повторения.

Но это часто даёт `O(n^2)`.

## Нормальный подход
Использовать `dict`:
- ключ = значение;
- value = количество повторений.

Тогда один проход по данным даёт решение за `O(n)`.

## Где это используется
- самая частая буква или слово;
- поиск большинства элементов;
- сравнение двух строк или списков по составу;
- анаграммы;
- статистика по событиям.

## Базовый шаблон
```python
freq = {}
for item in items:
    freq[item] = freq.get(item, 0) + 1
```

Это один из самых важных шаблонов во всём алгоритмическом курсе.

## Что можно делать после построения частот
- найти максимум по частоте;
- проверить, встречается ли элемент ровно `k` раз;
- сравнить две frequency maps;
- отсортировать элементы по частоте.

## Типичные ошибки
1. Считать частоты через вложенные циклы.
2. Плохо инициализировать первый элемент.
3. Забывать, что ключами должны быть hashable-значения.
4. Смешивать частоту и индекс.

## Практическое правило
Если задача звучит как:
- “сколько раз встречается...”
- “какой элемент самый частый...”
- “одинаковый ли состав...”

то почти наверняка нужен frequency map.
