# Неделя 4, День 3 - Дедупликация и lookup-паттерны

## Дедупликация
Дедупликация - это удаление повторов.

Часто задача формулируется так:
- оставить только уникальные элементы;
- удалить дубликаты;
- сохранить первый уникальный порядок.

## Lookup-паттерн
Lookup-паттерн - это быстрый ответ на вопрос:
- есть ли элемент;
- где он находится;
- встречался ли он раньше.

Для этого обычно используют:
- `set` для membership;
- `dict` для позиции или связанной информации.

## Два режима дедупликации
### 1. Просто уникальные значения
Если порядок не важен, обычно достаточно `set(items)`.

### 2. Уникальные значения с сохранением порядка
Тогда нужен проход слева направо + `seen`.

Шаблон:
- если элемент ещё не видели -> добавляем в результат;
- отмечаем элемент в `seen`.

## Где это полезно
- нормализация данных;
- обработка логов;
- уникальные id;
- повторяющиеся запросы;
- подготовка данных к дальнейшим алгоритмам.

## Типичные ошибки
1. Терять порядок там, где он важен.
2. Использовать список для `seen`, хотя нужен `set`.
3. Путать удаление дубликатов с подсчётом частот.
4. Хранить слишком много лишней информации в `dict`.
