# Неделя 2, День 1 - Алгоритм Евклида (НОД)

## Что такое НОД
НОД - это наибольший общий делитель двух чисел.

Примеры:
- `gcd(12, 18) = 6`
- `gcd(8, 14) = 2`
- `gcd(9, 28) = 1`

Если НОД равен `1`, числа взаимно простые.

## Зачем это нужно
НОД используется в задачах на:
- сокращение дробей;
- арифметику по модулю;
- проверку взаимной простоты;
- числовые преобразования;
- математические подзадачи в алгоритмах.

## Наивный подход
Можно перебрать все делители от `1` до `min(a, b)` и выбрать максимальный общий.

Но это медленно.

## Идея алгоритма Евклида
Главое наблюдение:

```python
gcd(a, b) == gcd(b, a % b)
```

То есть вместо перебора делителей можно многократно заменять пару чисел:
- `(a, b)`
- `(b, a % b)`

Пока второе число не станет `0`.

Тогда первое число и будет НОД.

## Пример вручную
Найти `gcd(48, 18)`.

1. `48 % 18 = 12` -> `gcd(48, 18) = gcd(18, 12)`
2. `18 % 12 = 6` -> `gcd(18, 12) = gcd(12, 6)`
3. `12 % 6 = 0` -> `gcd(12, 6) = 6`

Ответ: `6`.

## Почему это быстро
На каждом шаге числа заметно уменьшаются.
Поэтому алгоритм работает намного быстрее, чем полный перебор делителей.

Это один из классических примеров эффективного алгоритма.

## Итеративная форма
Обычно пишут так:

```python
while b != 0:
    a, b = b, a % b
return a
```

## Что важно учитывать
### 1. Порядок аргументов не важен
`gcd(18, 48)` и `gcd(48, 18)` дают один результат.

### 2. Ноль
- `gcd(a, 0) = a`
- `gcd(0, b) = b`

### 3. Отрицательные числа
В алгоритмических задачах обычно берут `abs(a)` и `abs(b)`.

## Типичные ошибки
1. Не понимать, почему используется остаток `%`.
2. Забывать обработку нуля.
3. Пытаться перебрать все делители вместо применения идеи Евклида.
4. Путать НОД и НОК.

## Практическое правило
Если в задаче встречаются:
- делимость;
- дроби;
- взаимная простота;
- общий шаг/период,

очень вероятно, что там нужен НОД.
